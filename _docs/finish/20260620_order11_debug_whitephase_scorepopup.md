# ホワイトフェイズ中のスコアポップアップ不具合修正

## 目標
現在のホワイトフェイズ中における、チェイン進行中のスコアポップアップ表示の不具合を修正する。

## 不具合内容
- スコア詳細表示時

1.チェイン中にスコアが加算されて表示される
2.チェイン終了・スコア確定時に、いきなりスコア数が増える

## 不具合の原因と修正方針
チェイン中に表示されるリアルタイムスコアと、チェイン終了時に確定するスコアに乖離が発生している原因は、表示側と確定側でスコア計算式が二重管理（ハードコード）されており、表示側にホワイトフェイズの特例（3乗化）が適用されていないためです。
対症療法的に表示側に計算式をコピペするのではなく、「スコア算出ルーチンの共通化」によって根本的に解決してください。

## 修正手順
### 1. スコア計算ルーチンの切り出し (`logic.js` または該当ユーティリティ)
* `logic.js` (または `score.js`) に、チェインごとのスコアを算出する純粋な関数（例: `calculateChainScore(chainCount, depth, phaseName)`）を新設（切り出し）し、export してください。
* この関数内部で、『PROJECT_MATH_AND_BALANCE.md』に定義されている「RATE」「連鎖ボーナス（通常2乗、ホワイトフェイズ時は3乗等の特例適用）」「階層ボーナス」のすべての計算を行い、最終的なスコア（BigInt等）を返すようにします。

### 2. 確定処理の置き換え (`logic.js`)
* `finalizeDestruction` 内で行っているスコア計算処理を、上記で新設した `calculateChainScore` を呼び出す形に置き換えてください。

### 3. 表示処理の置き換え (`ScreenEffects.js` 等)
* `showChainPopup` 等でリアルタイムのスコアを計算・描画する際、自前で計算するのではなく、上記で新設した `calculateChainScore` を呼び出し、返ってきた数値をそのまま表示するように修正してください。
