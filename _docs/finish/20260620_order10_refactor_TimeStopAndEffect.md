# 20260620_order10_refactor_TimeStopAndEffect

## 目標
現在の `GameState.isStasis` などに依存している時間停止関連のロジックをリファクタリングし、**「パズル時間停止フラグ」** と **「フェイズ進行時間停止フラグ」** に明確に分離する。
また、演出と時間軸の依存関係を明文化した資料 `PROJECT_EFFECT.md` を作成し、コンフィグ画面に安全なゲージ操作デバッグ機能を実装する。

## 実装内容 (Step 2.6)

### 1. GameStateの時間停止フラグ分離
`config.js` の `GameState` において、以下のフラグ（または同等の役割を持つプロパティ）を導入・整理する。
*   `GameState.isPuzzlePaused`:
    *   物理演算（Matter.js）、LIFE減衰、パズル画面上の演出（パーティクル、レーザー等）、パズルへのタップ操作を完全に停止させるフラグ。
    *   コンフィグ中や、フェイズ移行時のステイシス演出中などに `true` となる。
*   `GameState.isSystemPaused` (または同等のフェイズ進行停止フラグ):
    *   `PhaseManager` 等のフェイズ進行タイマーそのものを停止させるフラグ。
    *   コンフィグウィンドウ展開中のみ停止し、フェイズ演出中は進行させる。

### 2. PROJECT_EFFECT.md の新規作成
演出がどの「時間軸（パズル、フェイズ、システム現実時間）」に属しているかを定義する絶対資料を作成する。
記載内容は以下の項目を網羅すること。
*   エフェクト名
*   目的（1行説明）
*   描画レイヤー（例: `LAYERS.FOREGROUND_EFFECTS`）
*   時間軸（パズル、フェイズ、システム）
*   発火フラグ / トリガー
*   参照パラメータ

### 3. ロジックへの適用
*   `PhaseManager.js`:
    *   `update` 内のゲージ自然減衰処理は「パズル時間停止」中はスキップする。
    *   フェイズ移行演出のタイマー（`PHASE_WHITE_ENTER`の2000ms等）は「パズル時間停止」の影響を受けずに進める。
*   `ParticleManager.js`, `RippleManager.js`, `LaserEffect.js` 等:
    *   「パズル時間停止」中は、描画（`draw`）のみ行い、座標移動や寿命の減衰（`update`）をスキップする。
*   `physics.js`, `logic.js`:
    *   物理更新やLIFE減衰のスキップ条件を、新設した「パズル時間停止」フラグに紐付ける。

### 4. ConfigScene と デバッグUIの実装
*   コンフィグ画面内に、シフトゲージ・ブレイクゲージを任意（0, 500, 1000等）に変更できるデバッグボタンを追加する。
*   ボタン押下時は数値を変更して「フェイズ移行予約フラグ」を立てる。
*   コンフィグクローズ時にフラグをチェックし、フェイズ移行が予約されていれば、**パズルの時間停止を解除せずにそのまま `PhaseManager.enterWhitePhase()` などを発火させる**。これにより、演出が強制解除されてパズルが動いてしまうバグ（上書き競合）を回避する。

## 注意事項
*   この作業は既存の描画ループと更新ループに大きく影響を与えるため、リザルト画面やゲームオーバー演出などに意図しない停止が発生しないか慎重に検証すること。
*   作業完了後、いつものように `changelog.js` の更新、ドキュメントのインデックス同期、バージョン（ZZ）のインクリメントを行うこと。
