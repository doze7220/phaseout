# PhaseShift 演出ブラッシュアップ ロードマップ

## 目標
ノーマルフェイズ～ホワイトフェイズ開始～ホワイトフェイズ終了～ノーマルフェイズ、の一連の流れを実装するためのロードマップです。


## 実装フェイズ
### 【フェイズ1：背景マネージャーの実装】
現在、ゲームの背景は ScreenEffects.js や Visualizer.js などが各自の判断で描画しており、責務が散らかっています。
ここに新たに BackgroundManager.js（仮）という専用のモジュールを新設し、『PROJECT_ARCHITECTURE.md』で定義された「第1層（BACKGROUND）」の描画責務を完全に独立させます。
これがすべての土台となります。

### 【フェイズ2：既存エフェクトの削除・移行】
まずは「既存の第10層（前面）の波紋エフェクトを廃止し、すべて背景奥に押し込む」という大掃除が必要です。
現在 ScreenEffects.js 等にハードコードされているホワイトアウトや反転処理を削除し、新設したマネージャーへ権限を移譲します。これにより、以後の実装でバグが起きなくなります。

### 【フェイズ3：通常パズルでの背景実装】
土台が綺麗になったところで、第1層の最奥に「アステライアの静かな星空」を描画します。
この「静」の状態が美しく作れてこそ、後の「異常な事態」が際立ちます。

### 【フェイズ4：フェイズゲージ連動の背景演出】
星空の手前に「たゆたう同心円と純白への脱色（予兆）」を組み込みます。
フェイズ2で既存の波紋を消し去っているため、大量の宝石の隙間から「奥で何かが重々しく蠢いている」という究極のパララックス（奥行き）を、一切の描画競合なしで実装できます。

### 【フェイズ5：ホワイトフェイズ中の背景実装（開始〜終了）】
「純白の空間」と、設定資料にある「外側から内側に巻き戻る逆波紋（リバース）」のループアニメーションを実装します。

### 【フェイズ6：ホワイトフェイズ突入時の演出実装】
最後に「ゲージMAX到達」から「ホワイトフェイズ開始」までを繋ぐ、ステイシス（時間停止）を伴う「中央からの超巨大な純白波紋の爆発」を実装します。
状態Aと状態Bを完成させてから、その間を繋ぐトランジション（突入演出）を作ります。
