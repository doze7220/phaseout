# Boot画面（起動画面）のハイセンス化と世界観演出の追加

あなたは優秀なゲームエンジニアです。
プレイヤーがゲームを起動して最初に目にする `BootScene.js` において、世界観（システムによる管理と観測者の介入）を体現する、ハイセンスかつ低コストな起動演出を実装してください。

## 1. 修正対象ファイル
- `js/scene/BootScene.js`

## 2. 実装要件①：タイプライター風システム起動演出
現在のシンプルなロード待機画面に、サイバーパンク/システム起動風のタイポグラフィ演出を追加します。
- 起動直後にキャンバス全体を白(`rgba(255,255,255,1)`)にしてください。
- フォントは黒(`rgba(0,0,0,1)`)で表示してください。
- `update(deltaTime)` 内でタイマーを管理し、一定間隔（数フレームごと）に文字が1文字ずつ出現する「タイプライター風」のアニメーションを実装してください。
- 表示するテキストは、本作の世界観に合わせた以下のシステムメッセージ（例）とします。複数行に分けて表示するか、順次切り替えてください。
  - `> INITIALIZING ASTRAEA SYSTEM...`
  - `> CHECKING PHYSICAL FRAGMENTS... OK.`
  - `> AWAITING OBSERVER "GAZER" INPUT.`
  - (1行開けて)` PLEASE TOUCH SCREEN`
- `draw(ctx)` 内にて、小さく細いフォント（例: `14px monospace`）を用いてCanvas上に描画してください。
- `...` の部分はwaitを倍にして、ロード・処理をしている演出にしてください。

## 3. 実装要件②：タップ時のグリッチ遷移演出
ユーザーが画面をタップしてタイトルへ移行する際、単にクロスフェードさせるのではなく、システムが一瞬バグを起こしたような「グリッチ演出」を挟んでから TitleScene へ遷移させます。
- ユーザーがタップ（`handleInput`）した際、即座に遷移せず、状態を「グリッチ中（遷移ウェイト）」に切り替えます。
- 約100ms〜200msの間、テキストの描画に対して以下の Canvas 2D エフェクトを適用してください。
  - X座標をフレームごとにランダムに数ピクセルずらす。
  - 色収差（RGBシフト）：同じテキストを赤色（例:`rgba(255,0,0,0.8)`）とシアン色（例:`rgba(0,255,255,0.8)`）で指定し、本来の座標から左右に数ピクセルずらして描画（必要に応じて `globalCompositeOperation = 'lighter'` を活用）。
- グリッチ演出タイマーが完了した直後に、`SceneManager.changeScene(new TitleScene())` を実行して遷移してください。
