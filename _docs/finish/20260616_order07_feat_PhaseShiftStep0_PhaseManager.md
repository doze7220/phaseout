# フェイズシフト実装 Step0 状態管理マネージャーの新規実装

あなたは優秀なゲームエンジニアです。 
パズルゲーム『PHASE OUT ∴ Cluster Stirring』において、フィーバーモードである「フェイズシフト」の実装を目指します。

## 【目的】
フェイズシフト機能の土台となる状態管理マネージャーを新設し、既存のパズルロジックと安全に結線する「マイルストーン0」を実装します。
このステップでは、新しいゲームルール（無敵時間、ゲージ蓄積、演出等）は一切実装しません。純粋な「状態管理の箱」の作成と、「既存パズルがデグレードしていないことの担保」のみを目的とします。

## 【対象ファイル】
・新規作成: PhaseManager.js (システム・ロジック層に配置)
・編集: PlayScene.js
・編集: logic.js
・編集: Visualizer.js

## 【実装要件】
1. マネージャークラスの新設
PhaseManager.js を新規作成し、ステートマシンとして機能するクラスを実装してください。
管理すべきフェイズ状態として、「通常パズル時」「突入演出中」「ホワイトステイシス中」「解除演出中」の4つの状態を示す定数（文字列等）を定義してください。
内部に現在の状態を保持する変数を持ち、初期状態は「通常パズル時」としてください。
また、毎フレーム呼ばれる更新用メソッドと、現在の状態が「通常パズル時」であるかを判定するゲッターメソッドを実装し、シングルトンインスタンスとしてエクスポートしてください。

2. 既存ゲームループへの結線
PlayScene.js の更新処理 (update) 内で、PhaseManager の更新用メソッドを毎フレーム呼び出すように連携させてください。

3. ロジックへのガード条件追加
logic.js 内の「タップ入力による連鎖発火処理」および「時間経過によるLIFE減少処理」の冒頭に、PhaseManager のゲッターを利用したガード条件を追加してください。
現在の状態が「通常パズル時」である場合のみ、それらの処理が実行されるように制限をかけてください（現状は常に条件を満たします）。

4. デバッグ計器への可視化
Visualizer.js のデバッグ・オーバーレイ（第12層）の描画処理内に、PhaseManager が現在保持しているフェイズ状態の名称をテキストで1行描画する処理を追記してください。
ゲームの進行を邪魔しない位置（既存のFPS表示等の付近）に配置してください。

## 【注意事項】
・具体的な変数名や実装方法は、PROJECT_ARCHITECTURE.md の既存の設計思想に従って最適なものを選択してください。
・このステップの完了後、ゲームを起動して「今まで通りパズルが遊べること」「LIFEが減ること」「デバッグ画面に現在のフェイズが表示されていること」を必ず確認してください。
