# 実装指示書: デバッグ用設定値の定数化 (DebugConfig) とデバッグスタート機能の実装

## 目標
パズルプレイ中の状況操作や、特定フェイズからのテスト開始を迅速に行うため、デバッグ用パラメータを集約した新規ファイル `DebugConfig.js` を作成する。
現在 `ConfigScene.js` の DEBUGタブに直書き（マジックナンバー化）されているスコア倍率やゲージ操作の数値を `DebugConfig.js` に分離し、可読性と拡張性を向上させる。
合わせて、指定した初期状態からパズルを開始できる「デバッグスタートボタン」を第12層（DEBUG_OVERLAY）に配置する。

## 実装ステップ

### 1. [ADD] `js/core/DebugConfig.js` の新設
以下のデバッグ用設定オブジェクトを定義・エクスポートする。
*   **DEBUG_VALUES**: `ConfigScene.js` のボタンに適用する設定値リスト。「ボタンの表記名（label）」と「実際の数値（value）」を分離したオブジェクトの配列として定義する。
    *   例: スコア倍率 `[{ label: '1倍', value: 1 }, { label: '1万倍', value: 10000 }, { label: '1億倍', value: 100000000 }]`
    *   例: ゲージ操作 `[{ label: '0', value: 0 }, { label: '500', value: 500 }, { label: 'MAX', value: 1000 }]` 等
*   **START_PRESET**: タイトルからデバッグスタートした際のパズル初期状態定義（開始レベル、初期ライフ、各種ゲージ初期値等）。
*   **ENABLE_DEBUG_OVERLAY**: 第12層デバッグ機能の有効/無効を一括で切り替えるためのフラグ（将来の本番ビルド用）。

### 2. [MODIFY] `ConfigScene.js` （DEBUGタブ）のマジックナンバー排除
*   DEBUGタブのUI生成処理（レイアウトやボタンのインスタンス化自体）は既存の構造を維持する。
*   ボタンを生成する際にハードコーディングされていた「テキスト」や「適用する数値」を、`DebugConfig.js` からインポートした `DEBUG_VALUES` の配列を参照するように修正する。
*   これにより、テストしたい数値や倍率が変わった場合は `ConfigScene.js` のUIロジックを触ることなく、`DebugConfig.js` の値を書き換えるだけで済むようにする。

### 3. [MODIFY] 第12層への「デバッグスタート」ボタン統合
*   第12層（DEBUG_OVERLAY）を描画・管理している箇所（`Visualizer.js` の `drawDebug` 内など）にて、`DebugConfig.js` のフラグが有効な場合のみ「DEBUG START」ボタンを描画およびヒット判定登録する。
*   ボタンのヒット判定は `UIManager.js` に最上位レイヤーとして登録し、背面の通常タップ判定をブロックすること。
*   押下時は、通常のパズル遷移処理を実行しつつ、初期化完了直後に `START_PRESET` の値をゲーム状態（`GameState`, `PhaseManager` 等）にインジェクション（上書き）してパズルを開始させる。

## 実行にあたっての厳守事項
*   バージョンはパッチバージョンをカウントアップすること。
*   `ConfigScene.js` 内のボタンコールバック（実際の処理ロジック）が迷子にならないよう、データ駆動でUIそのものを自動生成するような過度な抽象化は行わないこと。
*   通常スタート時のゲームフローに一切の副作用を与えないようカプセル化を徹底すること。
