あなたは優秀なゲームエンジニアです。
パズルゲーム『Phase Out: Cluster Stirring』において、将来の機能拡張に備えた「コードベースのリファクタリング（大掃除）」を実行します。
`PROJECT_ARCHITECTURE.md`のルールに従い、機能や見た目を一切変更することなく、内部構造のみをクリーンアップしてください。

【対象ファイル】
・css/style.css
・js/entity/LaserEffect.js

【実装要件】
1. `css/style.css` の構造整理とブロック分け
・現在約640行に肥大化しているCSSの中身を、機能や役割ごとに論理的なブロックに分割し、コメントヘッダー（例: `/* ================= CSS Variables ================= */`）を挿入して整理してください。
・推奨するブロック構成:
  1. CSS Variables (Root定義)
  2. Reset & Base (全体設定, body, #game-container等)
  3. Layout (ヘッダー, パズル領域, フッターの大枠)
  4. UI Components (ボタン, スコア表示, タイマー, コンフィグモーダル等)
  5. Game Elements (LIFE/EXPゲージ, フローティングテキスト等)
  6. Animations & Effects (keyframes, ダメージフラッシュ, ステイシス効果等)
・※CSSの記述順序を整理するのみで、クラス名の変更やプロパティの削除・変更は絶対に行わないこと。

2. `js/entity/LaserEffect.js` のアーキテクチャ違反の解消
・`PROJECT_FUNCTION_INDEX.md` に記載の通り、現在 `LaserEffect#updateAndDraw` 内の `afterRender`（描画ループ）フックの中で、ゲーム状態である `burstFlag` 等を直接更新してしまっているアーキテクチャ違反を解消してください。
・【修正方針】: 
  エフェクトの進行状態（アニメーションの進捗やバーストフラグなど）は `GameState` のようなグローバル状態に書き込むのではなく、`LaserEffect.js` クラスの内部状態（インスタンスプロパティ）として完全にカプセル化してください。
・描画ループ（Canvasの処理）は「現在の状態を読み取って描画するだけ」の責務に純化させ、ロジック（システム層）に副作用を漏らさないように設計を修正してください。

【厳守事項】
・本ステップはリファクタリングです。ゲームの挙動、アニメーションの速度、UIの見た目、ファイルパスなどを一切変更しないでください。
・実行後、エラーが出ず、これまで通りレーザーが走り、UIが崩れていないことを確認すること。
