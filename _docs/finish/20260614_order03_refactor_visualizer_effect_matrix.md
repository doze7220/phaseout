# ビジュアライザ表現とエフェクト負荷の独立化（マトリクス設計）および「GLITCH」モードの追加

現在、ビジュアライザの表現（WAVE / BLOCK）と、描画負荷のエフェクト設定（FULL / LITE / NONE）の仕様が癒着しており、「LITEにすると強制的にBLOCKになる」といった不自然な動作になっています。
これを解消し、プレイヤーが「見た目」と「負荷」を独立して設定できるマトリクス構成へリファクタリングします。また、見た目の設定として新たに「GLITCH（グリッチ）」モードを追加します。

## 1. config.js の改修
コンフィグ設定において、以下の2軸を完全に独立させて定義してください。
- **エフェクト設定 (`AppConfig.EFFECT_LEVEL`)**: `'FULL'` / `'LITE'` / `'NONE'` (端末スペック用の負荷設定)
- **ビジュアライザ設定 (`AppConfig.VISUALIZER_MODE`)**: `'WAVE'` / `'BLOCK'` / `'GLITCH'` (見た目の好み)
※ビジュアライザは経験値効率（X座標）を示す重要なUIであるため、完全な「OFF」は廃止します。

## 2. Visualizer.js の改修
`Visualizer.js` における描画ロジック（`RenderStrategies` 等）を、上記2つの設定の組み合わせで判定するよう改修してください。

- **VISUALIZER_MODE === 'GLITCH' の場合（新規追加）**:
  - オーディオ（BGM）連動は行わず、WAVEモードと同じロジックでX座標（経験値効率）をベースにしたラインを描画する。
  - 時折、心電図のようにランダムなスパイク（波の跳ね上がり）を発生させる演出を組み込む。
  - `EFFECT_LEVEL === 'NONE'` の時は、WAVEモード同様に塗りつぶし（fill/加算合成）をスキップし、実線描画のみを行う。
- **VISUALIZER_MODE === 'WAVE' の場合**:
  - `EFFECT_LEVEL === 'NONE'` の時は、既存の超軽量ロジック（塗りつぶしをスキップし実線描画のみ行う）を適用する。
  - それ以外の時は通常のWAVE描画を適用する。
- **VISUALIZER_MODE === 'BLOCK' の場合**:
  - `EFFECT_LEVEL === 'NONE'` の時は、既存の `BLOCK_NONE` 相当の超軽量ロジック（FFT解析やランダム揺らぎをスキップする）を適用する。
  - それ以外の時は通常のBLOCK描画を適用する。

## 3. ConfigScene.js (UI) の改修
コンフィグ画面のUIを改修し、プレイヤーが「ビジュアライザ設定（WAVE / BLOCK / GLITCH）」と「エフェクト設定（FULL / LITE / NONE）」をそれぞれ独立したUI（既存のToggleSwitch等）で選択できるように変更してください。設定は正しく保存・適用されるようにしてください。
