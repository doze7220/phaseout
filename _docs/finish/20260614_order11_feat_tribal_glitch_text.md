# トライバル演出へのサイバーなテキストログ＆グリッチ表示の追加

## Goal
ロードマップ第3項目のブラッシュアップとして、新色アンロック時の「トライバル拡散演出」に、モノスペースフォントによるシステムログ表示（陣営名）を追加します。
出現時と、トライバルがフェードアウトし始めるタイミングの2回、サイバーパンク風の「グリッチ（RGBシフトと明滅）」を発生させて表示・消去を行うことで、システム（アステライア）が新しい理（ルール）を読み込んだカタルシスを演出します。

## 変更内容詳細

### 1. `js/core/config.js` の改修
*   **陣営データ（faction）の追加**:
    `COLOR_CONFIG` の各オブジェクトに、世界観設定に準じた以下の `faction` プロパティを追加してください。
    - Red: `IGNIS`
    - Orange: `HELIOS`
    - Yellow: `GAIA`
    - Green: `VERITY`
    - Cyan: `AETHER`
    - Blue: `CELESS`
    - Purple: `GNOSIS`

### 2. `js/render/ScreenEffects.js` の改修
*   **テキスト情報の保持**:
    `showTribalUnlockEffect(colorStr)` 内で、受け取った色から `faction` を取得し、以下のようなログテキストを生成してエフェクトオブジェクト (`this.tribalEffects`) に持たせてください。
    例: `text: \`- Infusion ${colorDef.name.toUpperCase()} palette : faction '${colorDef.faction}' -\``
*   **描画・グリッチロジックの追加 (`drawTribalEffects`)**:
    既存のトライバルシンボル描画処理の下（画面中央からやや下、`y + 80` 付近など）に、テキスト描画処理を追加します。
    - **フォント**: `16px monospace, "Courier New"`
    - **フェーズ管理**: 進行度（progress: 0.0〜1.0）に基づき、テキストの表示状態を以下の3フェーズに分割します。
        1. **出現グリッチ期（例: progress 0.0 〜 0.15）**:
           テキストが確定しきらないように、`Math.random()` で描画を時々スキップ（明滅）させつつ、赤(`rgba(255,0,0,0.8)`)とシアン(`rgba(0,255,255,0.8)`)の色でX座標をランダムに数ピクセルずらして複数回重ね描き（RGBシフト）し、バグったようにブレて出現させます。
        2. **通常表示期（例: progress 0.15 〜 0.75）**:
           白色（または対象の陣営色）でクリアに描画します。
        3. **消去グリッチ期（例: progress 0.75 〜 0.8）**:
           トライバルシンボルがフェードアウトし始める直前のタイミングで、再び強烈なRGBシフトと明滅（グリッチ）を発生させます。
        4. **消去（progress 0.8 以降）**:
           **テキストの描画を完全に停止（非表示）**します。これにより、シンボルがゆっくりフェードアウトしていく裏で、システムテキストだけが「バチッ」と唐突に消滅するエモーショナルな対比を表現します。

### 3. バージョニングとドキュメント更新
*   `changelog.js`: 新しいバージョン（例: `v0.20.4`）として「演出強化: トライバルアンロック演出に、陣営名を示すシステムログ表示を追加。出現時およびフェードアウト開始時にグリッチ演出を伴って表示・消去するよう実装」を追記してください。
*   `PROJECT_ARCHITECTURE.md`, `PROJECT_FUNCTION_INDEX.md`: 先頭のバージョン番号を同期して更新してください。
