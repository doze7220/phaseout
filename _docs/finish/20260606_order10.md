あなたは優秀なゲームエンジニアです。
パズルゲーム『Phase Out: Cluster Stirring』において、UI改修の最終段階である「Step3：ステータス背景オーディオビジュアライザの実装」を行います。
`PROJECT_ARCHITECTURE.md`の責務分離ルールに従い、Canvas描画ロジックとして実装してください。なお、このビジュアライザは「経験値（EXP）の進行度」とは完全に切り離され、純粋に「各色の破壊バランス（撹拌度合い）」を示すシステムメーターとして機能します。

【対象ファイル】
・index.html (ヘッダー背景用Canvasの追加)
・style.css (Canvasのレイアウト調整)
・config.js (UI設定トグルと波形状態の追加)
・effects.js (波形スパイク用APIの追加)
・logic.js (破壊時のスパイク呼び出し)
・renderer.js または Visualizer.js等の新規モジュール (ビジュアライザ描画)

【実装要件】
1. ビジュアライザ用Canvasの設置 (`index.html`, `style.css`)
・スコアやレベルが表示されている「ヘッダー（またはステータスパネル）」の背景に、ビジュアライザ描画専用の `<canvas id="visualizer-canvas"></canvas>` を追加してください。
・CSSで `position: absolute; z-index: -1;` 等を指定し、文字等のUI要素の背後に配置されるようにしてください。

2. コンフィグと状態定義 (`config.js`)
・`AppConfig` 等のコンフィグ設定に、描画モード設定を追加してください。
  `VISUALIZER_MODE: 'WAVE' | 'BLOCK' | 'OFF'` (デフォルトは 'WAVE')
・`GameState` に、各色ごとの「波形の振幅倍率（スパイク状態）」を管理するプロパティ `visualizerAmplitudes: {}` を追加し、全色の初期値を `1.0` としてください。

3. 破壊時のリアクティブな揺れ（スパイク処理）
・`effects.js` に `triggerVisualizerSpike(color)` を追加してください。
・`logic.js` の `finalizeDestruction` にて宝石が破壊された際、その色を引数にして上記関数を呼び出してください。
・呼び出された色の `GameState.visualizerAmplitudes[color]` を `5.0` などの高い値に跳ね上げさせます。

4. ビジュアライザ描画ロジック (`renderer.js` または描画ループ内)
`requestAnimationFrame` の描画ループ内で、以下の処理を実行します。
最初に `visualizerAmplitudes` の各値を毎フレーム `1.0` に向けて減衰（イージング）させてください。
その後、`GameState.colorDestroyCounts` を参照し、モードに応じた描画を行います。

　A. WAVEモード（リッチ・高負荷）
　・`globalCompositeOperation = 'lighter'` を使用し、現在アンロックされている各色の半透明なグラデーション波形（Sine波など、時間経過で横に流れるアニメーション）を重ねて描画します。
　・波形の「基本の高さ（振幅）」は、全色の中で最も破壊数が多い色を基準（100%）とし、それぞれの色の破壊数の割合（%）に連動させます。
　・その高さに対し、さらに `visualizerAmplitudes[color]` を乗算し、破壊時に激しく波打つようにします。
　・**【重要】** 全色が均等に破壊されている場合、すべての波形が同じ高さで重なり合い、加算合成によって「純白（ステイシス）」に発光するようなビジュアルにしてください。

　B. BLOCKモード（軽量・レトロ）
　・描画負荷を下げるため、パス描画や `lighter` を行わず、昔のオーディオコンポのような四角いブロック（`fillRect`）を縦横に並べるレトロなUIで表現します。
　・各色のブロックの高さ（積まれる数）を破壊数の割合とスパイク状態に連動させてください。

【遵守事項】
・Canvasの状態変更（`lighter`合成など）時は必ず `ctx.save()` と `ctx.restore()` を使用し、他の描画に影響を与えないこと。
・既存のSVGゲージ（LIFE/EXP）や文字UIのレイアウトを崩さないよう注意すること。
