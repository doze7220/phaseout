 実装指示書：ブラックフェイズ専用スコア表示のリッチ化・フォーマット修正

スプライト描画への統合は確認したが、UIのフォーマットと情報量がパズルゲームのカタルシスとして不十分であるため、以下の3点を修正せよ。

## 1. 桁数制限（12桁）の適用
フォントスケールの拡大に伴い、桁数が多すぎると画面外へ溢れてしまう。
*   `score.js` の `generateScoreData(value, maxDigits)` を呼び出す際、第2引数に `12`（または `LayoutConfig` 等で定義されているポップアップ用の制限桁数定数）を明示的に渡し、12桁＋単位で正しく省略表示されるように修正せよ。

## 2. チェイン数表記のフォーマット修正
チェイン数が生数字のまま描画されており、見栄えが悪い。
*   表示形式を、通常のスコアポップアップと同じにすること。
*   チェイン数を描画する際、単なる数字の文字列化ではなく、末尾に `" CHAIN"` などの接尾辞テキストを付与してから `ScoreRenderer.js` の `drawString` へ渡すこと。

## 3. 計算式（数式）の描画追加
スコアがどのように跳ね上がっているかを示すため、チェイン数とスコアの間にブラックフェイズ独自の「計算式」を描画せよ。
*   表示形式を、通常のスコアポップアップと同じにすること。
*   ブラックフェイズのスコア計算に導入した「チェイン数の平方根補正」を視覚的に表現するため、例えば `"√( " + GameState.blackHoleChainCount + " )^2 ="` あるいは `"√CHAIN"` といった文字列を構築し描画すること。
*   文字スケールはスコアより少し小さめ（例: `SCORE_SCALE` の半分程度）にし、画面中央に正しくセンタリングして配置すること。
*   ※必要であれば、`SpriteCacheManager.js` にて計算式に必要な記号（`√`, `(`, `)`, `=`, 等）のキャッシュ生成がサポートされているか確認・追加すること。
