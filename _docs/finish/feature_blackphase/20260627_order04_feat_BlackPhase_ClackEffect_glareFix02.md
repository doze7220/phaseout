# 実装指示書：ヒビ割れ画像の透過処理におけるCanvas最適化バグの回避（ダブルバッファリング実装）

ルミナンス変換でピクセルが完全に透明化されているにも関わらず、Canvasのシャドウレンダラーが画像外枠に影を落としてしまうブラウザ起因のバグ（putImageData後の最適化フラグ残存）を解消するため、転写用キャッシュを用いたダブルバッファリングを実装せよ。

## 1. SpriteCacheManager のルミナンス変換ロジック修正
`SpriteCacheManager.js` 内のヒビ割れ画像透過処理（ルミナンス変換）において、`putImageData` を行ったCanvasを直接キャッシュとして保持するのではなく、以下の手順で「新しいCanvasへの転写」を行うこと。

// 【修正要件（getImageData〜キャッシュ登録までの流れ）】
// 1. オフスクリーンCanvasA を作成し、元画像を描画して getImageData でRGBA配列を取得。
// 2. ルミナンス変換を行い、白を透明（Alpha 0）に、RGBを全て純黒（0）に書き換える。
// 3. 変換後のデータを CanvasA に putImageData で戻す。
// === ここから追加 ===
// 4. キャッシュ用として、同サイズの新しいオフスクリーン CanvasB を作成する。
// 5. CanvasB のコンテキストに対し、`drawImage(CanvasA, 0, 0)` を実行し、画像を転写する。
// 6. 最終的なキャッシュデータ（メモリ保持用）として、CanvasA ではなく「CanvasB」を採用する。

※この転写（drawImage）処理により、Canvasエンジンの不透明領域オプティマイザが強制的に再計算され、ドロップシャドウが正しく不透明ピクセル（ヒビ）のみに落ちるようになる。
