# 実装指示書：ブラックフェイズ中のタップ入力判定のバグ修正（早期リターンガード）

現状、ブラックフェイズ（PHASE_BLACK）中に宝石をタップすると通常のパズル破壊処理が実行されてしまい、かつブラックホールの脈動処理が不安定になるバグが発生している。これを修正するため、入力処理の最上流にてイベントをバイパスせよ。

## 1. `logic.js` の `pointerDownHandler` 修正
`js/core/logic.js` 内のタップ入力処理（`pointerDownHandler`）の**一番最初（最上流）**に、現在のフェイズがブラックフェイズであるかを確認するガード条件を追加せよ。

```javascript
// 【修正要件：pointerDownHandlerの冒頭】
if (PhaseManager.getCurrentPhase() === PHASE_BLACK) {
    // 1. 特異点パルスの付与（GameState.blackHoleVisualPulse 等の加算）を実行する。
    // 2. ブレイクゲージ（寿命）のタップ回復処理を実行する。
    // 3. 即座に `return;` (または該当する終了処理) を行い、これ以降の宝石のヒット判定、
    //    startChain、LIFE減少判定などを完全にスキップ（無効化）せよ。
}
```

※これにより、ブラックフェイズ中は「画面のどこをタップしても100%確実にブラックホールが脈動し、宝石の破壊や連鎖は物理的に発生しなくなる」状態を確約すること。
