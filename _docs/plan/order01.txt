実装仕様書：物理連鎖プロトタイプ
1. 開発環境・技術スタック
言語: HTML5, JavaScript (ES6+)
物理エンジン: Matter.js
描画: Canvas API
2. 物理空間・オブジェクト設計
① ワールド設定
画面中央に 400x600 程度の「容器（静止した壁）」を作成。
重力は標準（y方向）。
② 宝石（Jewel）オブジェクト
形状: 円形（初期プロト用。後に矩形・三角を追加可能にする）
サイズ: 半径 15px 〜 20px のランダム
属性（Dataプロパティ）:
colorId: 0〜6 の整数（7色）
shapeId: 形状識別子
物理プロパティ:
摩擦(friction): 0.05
反発係数(restitution): 0.3
密度(density): 0.001
3. コアロジック実装
① BFS連鎖探索（核心）
タップされた宝石 A を起点に以下の探索を行う：
TargetColor = A.colorId を取得。
A に現在「衝突（Collision）」しているオブジェクトのリストを取得。
リスト内のオブジェクトのうち、「未訪問」かつ「colorId == TargetColor」のものをキューに追加。
全ての接続された同色オブジェクトが特定されるまで再帰的に繰り返す。
結果のリスト（連鎖リスト）を返す。
② 破壊処理
連鎖リストの個数 n を確認。
n >= 3 の場合：
対象オブジェクトをワールドから削除。
削除された座標にパーティクル演出を生成。
スコア加算： score += Math.floor(Math.pow(10, (n - 3) * 0.5) * 100)
n < 3 の場合：
削除はするがスコアは加算しない（ペナルティ的演出）。
③ 補充システム
宝石が削除された後、容器の上部（y < 0）のランダムなx座標から、削除された数と同数の新しい宝石をスポーンさせる。
4. 演出・UI
① 伝播エフェクト
タップ時、連鎖リストに含まれる宝石同士を、中心点を結ぶ「光の線（白または同色の発光ライン）」で 0.2秒間描画する。
② スコア表示
画面上部に Score を表示。
インフレを体感させるため、加算時は「数字がドラムロールのように回る」あるいは「急激にカウントアップする」演出を行う。
5. AIへの初期指示プロンプト例
「Matter.jsを使って、上記のBFS連鎖探索ロジックを持つHTML1ファイルのプロトタイプを作成してください。 まずは『7色の円が降り積もり、クリックすると同色の接触している円が連鎖して消える』という基本機能を実装して。 その後、連鎖数に応じたスコア表示と、消えた分が上から降ってくる補充機能を追加してください。」
