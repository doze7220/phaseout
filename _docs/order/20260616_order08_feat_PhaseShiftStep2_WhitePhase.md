# PhaseManager 更新計画：Step 2 (ホワイトフェイズのロジック、リバースリンク、および終了フェイズの実装)

本タスクでは、v0.26.4で土台が完成したフェイズシフトに対し、フィーバーモード（白の世界）としての絶対ルール、黒の領域への導線となる「リバースリンク」、および通常パズルへ戻るための「終了フェイズ（EXIT）」を実装します。

## 1. 白の理（無敵モードとインフレの適用）
PHASE_WHITE 滞在中は、以下の特別ルールを適用してください。
*   **タイマー管理とコンフィグ化 (`config.js`, `ConfigScene.js`, `PhaseManager.js`)**:
    *   `config.js` の `AppConfig` に、ホワイトフェイズの継続時間（デフォルト30秒）を追加してください。
    *   `Config.js` でこの秒数を任意に変更・保存できるようにしてください。
    *   `PhaseManager.update` にてカウントダウンを行い、時間切れで `PHASE_WHITE_EXIT` へ移行させてください（※即座にNORMALへは戻さないこと）。
*   **LIFE減少の停止 (`logic.js`)**:
    *   時間経過によるLIFE自然減少、およびタップ時のLIFE消費を `0` にしてください。
*   **経験値ペナルティの撤廃 (`logic.js`)**:
    *   通常時の「経験値の色別減衰ペナルティ」計算を無効化し、無条件で最高効率の経験値を獲得できるようにしてください。
    *   一時的に、全経験値効率を100%に上書きしてください。裏では通常の全経験値効率計算は行っていても問題ありませんが、ビジュアライザに反映するのは、この一時的な経験値効率を反映してください。現在補正値と別にホワイトフェイズ中補正値を用意し、ホワイトフェイズに100%までの差分を入力する、などの対応でも良いです。
    *   チェイン数による減算補正はスキップしてください。
*   **スコアの三乗インフレ (`logic.js`)**:
    *   スコア計算における連鎖ボーナスを `(チェイン数 - 2) ^ 2` から `(チェイン数 - 2) ^ 3` に変更してください。
*   **UIの強調表示 (`ScreenEffects.js` または `ScoreRenderer.js`)**:
    *   数式ポップアップ表示において、インフレの証である `^3` の部分だけを赤く発光させ、フォントサイズを意図的に大きくする等の強調描画を行ってください。

## 2. 黒への導線（リバースリンクの解禁）
PHASE_WHITE 滞在中のみ、以下のルールで連鎖アルゴリズムと演出を上書きしてください。
*   **順方向リンクの封印と逆順の解禁 (`ChainAlgorithm.js`)**:
    *   通常のスペクトル順（赤→橙→黄…）の接続を**完全に封印**してください。
    *   代わりに、逆順（紫→青→水…）の接続のみを許可する「リバースリンク」を有効化してください。
*   **リバースリンク専用演出 (`LaserEffect.js`, `SoundManager.js`, `renderer.js`)**:
    *   **SE**: リバースリンク成立時、専用の不協和音（システムエラーや逆回転を思わせる音）を再生してください。
    *   **フラッシュ**: リバースリンクの起点となった宝石が一瞬だけ黒く染まる処理を追加してください。
    *   **レーザー**: だんだんと黒色に近づくように（あるいは漆黒のレーザーとして）描画してください。※描画時の `globalCompositeOperation` を `source-over` や `difference` 等に適切に切り替えて黒を表現すること。

## 3. 白の空間の暫定描画（背景の鼓動）
将来的な背景演出の本格実装までの暫定対応として、PHASE_WHITE 中は以下の背景描画を行ってください。
*   **白の鼓動 (`Visualizer.js` または背景レイヤー)**:
    *   `PhaseManager` が `PHASE_WHITE` の間は、第1層の最奥の背景全体を白（#FFFFFF）で塗りつぶし、`Math.sin` 等の関数を用いて「白」から「薄灰色」へと心臓のように重々しく明滅（鼓動）するようなアニメーション描画を行ってください。
    *   **※注意事項**: プレイヤーの視線誘導の観点から、通常のビジュアライザ波形（WAVEやBLOCK）の描画処理は停止・スキップせず、この鼓動する白い背景の上にそのまま重ねて描画し続けてください。

## 4. 白の終焉（PHASE_WHITE_EXIT の実装と通常への帰還）
PHASE_WHITE のタイマー終了時、即座に通常のパズルへ戻るのではなく、以下の手順でトランジション（終了の儀式）を挟んでください。
*   **ステイシスの発動と無敵ルールの解除 (`PhaseManager.js`)**:
    *   `PHASE_WHITE_EXIT` へ移行した瞬間、再び `timeScale = 0` と `isStasis = true` を適用し、物理演算とタップ入力を完全停止させてください。
    *   このタイミングでLIFE減少やインフレ等の「白の理」をすべて解除してください。
*   **BGMのぶつ切りストップと無音ステイシス (`SoundManager.js`)**:
    *   ステイシス移行と同時に、再生中のフェイズ専用BGMをフェードアウトさせるのではなく、**即座に完全停止（ぶつ切りストップ）**させてください。
    *   ステイシスが解除されるまでの間、BGMが一切鳴らない「完全な無音（静寂）」の状態を維持してください。
*   **白フラッシュと世界（BGM）の再起動 (`ScreenEffects.js`, `SoundManager.js`, `PhaseManager.js`)**:
    *   無音状態のまま終了の合図として、画面全体に純白のフラッシュ演出（`triggerWhiteFlash` 等）を発火させてください。
    *   フラッシュ演出等の時間（例: 1秒）が経過した後、`PHASE_NORMAL` へ移行してステイシスを解除してください。
    *   **通常パズル復帰と全く同じタイミングで、通常のステージBGMを「最初から（再生位置0秒で）」再生開始**してください（※世界が再起動したような演出になります）。
