# 実装指示書：PHASE_WHITE_ENTER 演出シーケンスの実装

## 目的
シフトゲージMAX到達による「ホワイトフェイズ（フィーバー）」突入演出を実装する。
ただの白フラッシュではなく、「完全な無音・停止」→「トライバルとログの顕現」→「図形の極大膨張による画面の白塗り（イン）」→「真理の瞳が開くような透明円形ワイプと追従する波紋（アウト）」というトランジションを実現する。

## 前提条件（アーキテクチャの絶対厳守）
*   演出の進行度（`elapsed`）は、**必ず `realDelta`（システム実時間）を加算して管理すること**。パズル時間（`gameDelta`）や絶対時間（`performance.now()` 等）へ依存してはならない。
*   描画はすべて `ScreenEffects.js` などのCanvas上で行い、DOM操作は厳禁とする。
*   【重要】各演出フェーズ（ステイシスのタメ、トライバル展開、極大膨張、波紋の収束など）の所要時間（ミリ秒）はコード内にマジックナンバーとして直書きせず、必ず `config.js` の `EFFECT_MATH_CONFIG.PHASE_WHITE` 等として定数化し、将来の尺調整や短縮版実装を容易にすること。

## 演出タイムラインと実装ステップ

### 1. ステイシスと無音化
*   **発火:** `PhaseManager` が `PHASE_WHITE_ENTER` に移行した瞬間。
*   **状態:** `isPuzzlePaused = true` となり物理演算が停止。
*   **サウンド:** `SoundManager` によりBGMを停止させる。
*   **インプット:** コンフィグを含め、画面タップを抑制する。将来的にはタップで演出スキップできるよう検討するが、現時点では未実装とする。

### 2. アステライアトライバル展開 ＆ ログ表示
*   **描画先:** Canvas 2D（第8層 または 最前面エフェクト層）。
*   **トライバル描画:** 画面中央から7つの円を展開（Expand）し、定位置に到達した瞬間に強い発光（Ignite: `shadowBlur` や線幅の拡大）を伴って定着させる。トライバル描写方法については後述。
*   **システムログ描画:** トライバル発光と同時（または連動）に、以下のテキストを画面中央に **センタリング（`textAlign = 'center'`, `textBaseline = 'middle'`）** で描画する。等幅フォント（monospace等）を使用し、システムチックに表示すること。表示タイミングを行ごとにConfig.jsで定義し、制御できるようにすること。
    ```text
    SPATIAL POSSIBILITY FRAGMENTS : CRITICAL
    AVERAGING EXISTENCE PROBABILITIES...
    INITIATING PHASE TRANSITION...
    
    [ PHASE SHIFT ]
    
    " WHITE STASIS "
    ```

### 3. 大膨張トランジション・イン
*   トライバル発光から数フレームのタメの後、**トライバルの円の半径（`radius`）と線幅を爆発的な速度で極大膨張**させる。
*   円の境界が画面の端を越えるまで広がり、**「図形そのものによって画面全体が純白に塗りつぶされた状態」**をトランジションの頂点とする。
*   具体的なステップは次の通り。
    1. 前ステップで描画したトランジションをそのまま利用する。トライバルが外円から加速度（最初は高速→低速）を付けて広がる。一番内側の円は同時に内円を縮小させて内側を塗りつぶす。
    2. 最後の円が画面を覆い尽くしたら一瞬だけウェイトを入れる。

### 4. 多重波紋トランジション・アウト
*   画面が純白になった直後、画面中央から **透明な円形の穴（`globalCompositeOperation = 'destination-out'` を用いた `arc` 描画）** を爆発的に広げ、下層の「白飛び・白黒反転した新世界（背景）」を露出させる。
*   具体的なステップは次の通り。
    1. 中心点から円形にワイプを入れ、外側に広がる。そのさいに波紋のように、白円（幅大小あり）がいくつか追従して広がる。
    2. 追従波紋が完全に画面外に消えたらトランジション終了。

### 5. PHASE_WHITE 開始
*   波紋の最後の一波が画面外へ消え去った瞬間に、`PhaseManager` を `PHASE_WHITE` へ本格移行させる。
*   **状態:** `isPuzzlePaused = false` となりパズルが再始動。
*   **サウンド:** `SoundManager` により専用のフィーバーBGM（`phase_shift` 等）を頭からフルボリュームで再生開始する。

## 2.トライバル描写
1. [展開 - Expand] (progress 0.0 → 0.5)
2. 画面中央の1点から、純白の細い線で描かれた「7つの円」が、360度を7分割した角度（約51.4度ずつ）に向かって、スッ…とイージング（Ease-Out）を伴って広がっていく。ラインの太さは4px。
3. 7つの円がそれぞれの定位置（綺麗な対称図形）に到達してピタッと止まる。
4. 止まった瞬間、細かった円のライン（lineWidth）が力強く太くなり、透明だった線が**「純白（あるいは7色のグラデーションから純白へ）」へとジワッとフェードで着色・発光（shadowBlur）**する。各ラインの隙間は2px開けて止まる。

## 実行要件
以上のタイムラインを `PhaseManager.js` のタイマー管理と `ScreenEffects.js` の描画ロジックを連携させて実装しなさい。コードは可能な限りカプセル化し、各ステップの境界が明確になるよう設計すること。
