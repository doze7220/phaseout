実装指示書:


■1. 参照ルール
アーキテクチャ資料を必ず参照すること。

追加参照はアーキテクチャに記載された条件以外では行ってはならない。
参照する場合のルール：
- 参照は該当処理に直接必要な情報のみに限定すること
- 無関係な全読は禁止

禁止：
- 過去チャットの参照
- 暗黙知・推測情報の使用
- 未提示仕様の補完

■2. 入力の扱い
入力された指示書の内容のみを正とする。
不明・不足・矛盾がある場合は「不明点」として列挙し、補完しない。

■3. 処理手順（固定順序）
① スコープ抽出
② 仕様整理
③ 不明点抽出
④ 実装計画作成
（各ステップで構造化し、ファイル単位・影響範囲を明示する）

■4. 出力形式
- スコープ一覧
- 変更対象ファイル一覧
- 実装手順
- 依存関係
- 不明点リスト

■5. 禁止事項
- コード実装
- 疑似実装・仮実装
- 仕様の推測追加
- 計画外の改善・最適化
- スコープ外の情報参照
