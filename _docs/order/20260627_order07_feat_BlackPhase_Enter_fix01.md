# 実装指示書：ブラックフェイズ突入演出のビジュアル・ブラッシュアップ

テスト実装の結果、Canvasアニメーションおよび宝石の描画に想定との齟齬が確認された。添付したイメージ画像（赤ラインでの鏃指定）および以下の指示に従い、演出を修正せよ。

## 1. トライバル（七重円）のサイズと収縮アニメーションの修正
現在、画面外にはみ出すほど巨大に描画されているトライバル円を修正する。
*   一番外側の円（最大半径）を、ブラックホールの初期サイズ（`BLACK_HOLE_RADIUS_MAX` 等）と同一にせよ。
*   アストライアのトライバル設定（外縁300、内縁40）の比率を維持したまま、特異点（BH）サイズへ圧縮するアニメーションを実装する。
    *   **外縁の固定:** 一番外側の円の半径（外縁）を `TRIBAL_RADIUS_OUTER` （100）に固定せよ。
    *   **内縁の固定:** 一番内側の円の半径（内縁）は、 `TRIBAL_RADIUS_INNER` （14）として動的計算せよ。
    *   **隙間の固定と線幅計算:** 外縁と内縁の間に7つの同心円を描画する際、**円と円の間の隙間（間隔）は必ず「4px」に固定**すること。
    *   最終的な描画手順は `WHITE_PHASE_ENTER`か`WHITE_PHASE_EXIT`を参照せよ。
        *   **【重要：描画仕様の差異】** これらのホワイト演出では一番内側の円がドーナツ状となっているが、今回はブラックアウトからのワイプとなるため、一番内側の円をドーナツ状ではなく「円盤状（黒塗りつぶし）」とすること。
*   アニメーション（`WIPE_IN`）時は、画面外からこの「厳密に計算されたBHサイズの7重円」に向かって収縮していく挙動とすること。

## 2. 十字の楔（鏃）のパスデザイン修正
添付画像（赤ライン）の通り、鏃（やじり）のデザインを「V字の切れ込みを持つ鋭角な形状」へ変更せよ。
*   中央のBHサイズの円の縁（半径R）を「鏃尻の凹み部分の頂点」とする。
*   そこから上下左右の4方向に対して、外側に広がる2点と、最も外側に突き出る先端の1点を結ぶパス（`lineTo`）を構築すること。
*   （例：上方向なら `(cx, cy - R)` を起点とし、左斜め上、中央最上部、右斜め上を経由して起点に戻る。これを4方向分描画して `fill` および `stroke` を行う）

## 3. ブラックステイシス宝石の「ハードライト（hard-light）」合成への変更
`SpriteCacheManager.js` の宝石黒化処理（`isBlackPhase` 時）において、`globalCompositeOperation = 'source-atop'` で黒を塗っている処理を破棄せよ。
*   代わりに `globalCompositeOperation = 'hard-light'`（ハードライト合成）を使用し、暗色（例：`rgba(30, 30, 30, 0.8)` 等）を合成すること。
*   これにより、ベースの宝石が持つハイライトとシャドウの立体感を維持したまま、美しく黒ずんだ状態を作り出すこと。

## 4. ブラックアウト時のヒビ（亀裂）消去
*   ホワイトフェイズから `PHASE_BLACK_ENTER` に突入し、ブラックアウト（フリッカー）が始まった段階で、画面に描画されていた「ブレイクゲージのヒビ割れ（亀裂）」を完全に消去（描画停止またはクリア）すること。
*   暗転およびログ表示中に、ヒビが浮いて描画されないよう厳格に排他制御せよ。

## 5. マジックナンバーの禁止
*   アニメーションで用いる各種定数・係数・パラメータは、必ずeffectConfig.jsにてコメント付きで設定すること。
