# 実装指示書：フルリンク達成後のプリズムリンクUIループ上書き演出

## 目的
フルプリズムリンク（7色）を達成した後も連鎖が継続した場合の「オーバーリンク」の熱狂を視覚化するため、P-Link UIのスタンプ点灯アニメーションが2周目、3周目と既存アイコンを上書きしてループ発火する機構を実装する。

## 変更対象ファイル
*   `js/render/ScreenEffectPopup.js` (P-Linkステップ点灯ロジックの改修)

## 実装手順

### 1. `triggerPrismLinkStep` の無限ループ対応
*   `ScreenEffectPopup.js` 内の `triggerPrismLinkStep(step, baseColorId, ...)` メソッドを改修する。
*   現在7回目までしか想定していない処理を撤廃し、`step` が8以上の場合でも処理を継続させる。
*   配列の対象インデックスをモジュロ演算を用いて計算する。
    例：`const iconIndex = (step - 1) % 7;`
    ※ホワイトフェイズ中のリバースリンク（逆走）時も、モジュロによって正しく右端からループを繰り返すようインデックス計算を維持すること。

### 2. 周回数（loopCount）の保持とアニメーションの再点火
*   対象のアイコンオブジェクトを取得（または生成）した際、以下の状態を更新する。
    1.  `loopCount` プロパティに現在の周回数を設定する。
        例：`icon.loopCount = Math.floor((step - 1) / 7);`
    2.  落下・フラッシュ等の点灯アニメーションを管理するタイマー（`elapsed`等）を `0` に強制リセットする。（これにより、既に点灯済みのアイコンが再び激しくバウンドして上書きされる）

### 3. 周回数に応じたオーバードライブ描画（drawPopups内部）
*   `drawPopups` 内のP-Link UI描画処理にて、アイコンごとの `loopCount` を参照し、周回を重ねるほど発光が激しくなるよう Canvas コンテキストを操作する。
    *   **loopCount === 0 (1周目):** 通常描画（既存ロジック通り）。
    *   **loopCount === 1 (2周目):** 加算合成 (`ctx.globalCompositeOperation = 'lighter'`) を適用し、スケールを僅かに拡大（例: 1.1倍）して激しく白飛びさせる。
    *   **loopCount >= 2 (3周目以降):** さらに、先ほど実装した虹色グレア（`ctx.shadowColor = 'hsl(...)'`, `ctx.shadowBlur = 15`）を適用し、インフレの限界突破を表現する。
*   ※ 描画コンテキストは必ず `ctx.save()` / `ctx.restore()` で保護すること。

## 前提条件・アーキテクチャ厳守
*   すでに実装済みの「フルリンク時のアステライア昇華演出」のトリガー（スコア確定時）とは競合しない。本処理は連鎖が「伸びている最中」のステップアニメーション描画のみの改修である。
