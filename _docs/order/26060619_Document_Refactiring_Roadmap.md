# ■ ドキュメント整備タスク一覧
1. 構造整理（設計レベル）
- 章構成の重複（例：レンダーパイプライン説明の二重化）を統合する
- 「仕様」「説明」「禁止事項」のセクション分離を明確化する
- レイヤー定義（12層）とイベントフロー説明の重複を整理する
- GameState / Scene / PhaseManager の役割境界を明文化する

2. 冗長性削減（文章レベル）
- 各モジュールの共通説明（DOM禁止・ロジック禁止など）を共通ルールへ集約
- 同一意味の禁止事項の重複削除
- 長文化している責務説明の圧縮
- 表の説明と本文の重複削除

3. 用語・表記統一
- Phase / フェイズ表記の統一
- Scene / シーン表記の統一
- ステート表現（State / 状態）の統一
- 同義語の揺れ（例：フェイズシフト/PhaseShift）の整理

4. 表・データ構造の整理
- スコアフォーマット表の分離（UI仕様として独立章化）
- モジュール責務表のフォーマット統一
- 重複しているレイヤー説明表を1つに統合

5. 仕様の明確化（曖昧領域）
- PhaseManager / SceneManager / GameState の優先順位ルール追加
- イベントフローの責任範囲（誰が発火・誰が処理するか）明文化
- STASIS / GAMEOVER時の停止範囲の明確化
- destroy時のリソース解放ルール追加（未定義部分の補完）

6. 安全性・事故防止ルール追加
- 変更禁止領域（アーキテクチャ・イベント順序）の再強調
- AI編集可能範囲の明文化
- 「意味変更禁止」の適用範囲明確化
- 差分レビュー前提の編集フロー定義

7. 運用改善（AI-IDE向け最適化）
- AI用・監査用・編集用の役割分離セクション作成
- 「安全リファクタリング対象ルール」の明文化
- ドキュメント更新フロー（誰がどのAIで何をするか）定義
- 変更時チェックリスト追加
