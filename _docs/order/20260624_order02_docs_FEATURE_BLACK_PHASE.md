# 依頼：ブラックフェイズ専用アーキテクチャ資料の作成

ブラックフェイズのコアループ実装、お疲れ様。
今後、このブラックフェイズに対して「全宝石の吸い込み」「専用のシステムログ演出」「特異点でのアーティファクト攻防」など、大量の機能追加と演出強化を行っていく予定だ。

その際、毎回巨大な `PROJECT_FUNCTION_INDEX.md` 等を検索し直すのは非効率なので、現在の実装状態をまとめた特設ドキュメントを作成したい。

現在のチャットのコンテキストと、先ほど実装した内容を元に、プロジェクトルートに `FEATURE_BLACK_PHASE.md` というマークダウンファイルを作成し、以下の構成で詳細なアーキテクチャと現状の実装状態を書き出してくれ。

## ドキュメント構成案（FEATURE_BLACK_PHASE.md）

1. **変更ファイルと対象関数一覧（インデックス）**
   - 今回の実装で変更・追加を行ったファイル名と、その中のどの関数・メソッドで何の処理を追加したかをリスト化すること。
   - （※将来 `PROJECT_FUNCTION_INDEX.md` 等へそのまま移植しやすいフォーマットで列挙すること）

2. **ブラックフェイズ概要とステート遷移**
   - 発動条件と、`PhaseManager.js` における `ENTER` -> `BLACK` -> `EXIT` の遷移フローとタイマー仕様。

3. **関連する定数と状態（GameState）一覧**
   - `config.js`, `effectConfig.js` に追加した定数と、`GameState` のプロパティ（`blackHolePulse` 等）の一覧と役割。

4. **操作ロジックと描画の仕組み**
   - `logic.js` 等で行っているタップ制限、連鎖キャンセルのガード処理と、特異点タップ時の脈動加算ロジック。
   - `ScoreRenderer.js` によるヘッダーのヒビ割れ描画と、`BackgroundManager.js` による特異点描画・フェードの仕組み。

5. **今後の実装予定（To-Do）**
   - 今回は仮実装として見送ったが、今後追加予定のリスト（専用のホラーログ演出、テクスチャ画像への差し替え等）をメモしておくセクション。

6. **重要**
   - 今後の新チャットでも当ファイルを読めば、迅速に作業ができる情報を整えたい。当指示書に記載のある情報以外で必要なものがある場合は、資料を作成する前に提案すること。
