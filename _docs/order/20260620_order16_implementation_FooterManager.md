# 実装指示書: 仮キャラクターUIマネージャー（NO SIGNAL表示）の実装

## 目標
パズル画面下部（フッター領域）の隙間を埋め、将来のキャラクター描画マネージャーの土台となる `FooterUIManager.js` を新設する。
現段階では「未編成」のプレースホルダーとして、フッターを横に3分割した枠に「NO SIGNAL」のサイバーパンク風モニターパネルをCanvasで描画する。
描画負荷は `AppConfig.EFFECT_LEVEL` に応じて3段階で制御すること。具体的な変数名や描画アルゴリズムは、既存のアーキテクチャと最も調和する形でAI-IDEが適切に設計・実装すること。

## 実装ステップ

### 1. [MODIFY] `LayoutConfig.js` の拡張
* フッター領域に関するレイアウト定数（フッター全体の高さ、外枠のパディング、3分割パネル間のギャップなど）を管理するセクションを新設する。

### 2. [ADD] `js/render/FooterUIManager.js` の新設
以下の要件を満たすクラスを新規作成する。
* **初期化**: 既存のマネージャーに倣い、`MasterRenderer` の「第7層（UI_BASE）」に自身の描画メソッドを登録する[i]。
* **レイアウト分割**: `LayoutConfig.js` の値を参照し、フッター領域を横に3等分する座標計算を動的に行う。
* **パネル描画（NO SIGNAL）**: 計算された3つの領域それぞれに対し、以下の要件でモニターパネルを描画する。
  * 背景は深い黒系とする。
  * 中央にシステムフォントで「NO SIGNAL」等のテキストを配置する。
  * **エフェクト負荷制御（EFFECT_LEVEL連動）**[ii]:
    * `FULL`: 走査線（スキャンライン）の描画、透明度のランダムな明滅、および低確率でのグリッチ（色収差や座標ズレ）を組み合わせたリッチなノイズ表現を行う。
    * `LITE`: 走査線とグリッチ計算を省略し、透明度のランダムな明滅（システム稼働感）のみを適用する。
    * `NONE`: 乱数計算を一切行わず、固定アルファ値による完全な静止画として最軽量で描画する。

### 3. [MODIFY] システムへのフック
* `effects.js` 等の適切な初期化フロー内で `FooterUIManager` をインスタンス化し、システムへ組み込む。

## 実行にあたっての厳守事項
* バージョンはパッチバージョンをカウントアップすること。
* DOM操作は絶対に行わず、すべてCanvas 2D APIによる描画で完結させること[iii]。
* 描画メソッド内では必ず `ctx.save()` と `ctx.restore()` をペアで使用し、クリッピング、透明度、フォント設定などのコンテキスト変更が他レイヤーへ絶対に漏洩しないよう完全にカプセル化すること[iv]。
