# フェイズシフト実装 Step1 ゲージロジック・予兆波紋・発動演出の実装

あなたは優秀なゲームエンジニアです。
パズルゲーム『PHASE OUT ∴ Cluster Stirring』において、フェイズシフトの「見えないゲージの攻防」および「大当り発動時の演出」を実装するマイルストーン1を実行します。

## 【目的】
PhaseManager内にフェイズゲージの蓄積・減衰ロジックを組み込み、その数値を監視するためのデバッグ計器を追加します。また、ゲージ量に応じた予兆演出（波紋）と、限界突破時の大当り演出（ステイシスとホワイトアウト）のタイムラインを実装します。
※ホワイトステイシス中の特別ルール（無敵時間、リバースリンク、経験値100%等）はStep2以降で実装するため、本ステップでは「状態の遷移と演出の再生」までをゴールとします。

## 【対象ファイル】
・編集: PhaseManager.js
・編集: logic.js
・編集: Visualizer.js
・編集: effects.js / ScreenEffects.js (波紋・白フラッシュ描画用)
・編集: SoundManager.js (BGM頭出し再生用)
※必要に応じて config.js に定数を追加してください。
※原則としてマジックナンバーは厳禁です。どうしても必要な場合はユーザーに確認を取ること。

## 【実装要件】

### 1. フェイズゲージの蓄積と加速減衰ロジック (PhaseManager.js)
- 内部に「フェイズゲージ現在値」と「臨界値（MAX値）」を定義してください。
    - 臨界値の初期値は1000を仮設定とする。
- ゲージを加算するメソッドを作成し、logic.jsの連鎖完了時（経験値獲得等のタイミング）に、連鎖数などを基準としたポイントを加算できるようにしてください。
    - プリズムリンク時のみ、加算すること。
    - プリズムリンク時のチェイン数、階層数に応じて数値が上昇すること。
    - 平均して5回程度でフェイズシフトが発動する値に設定すること。
    - それらの計算式は一時委ねるが、詳細は PROJECT_MATH_AND_BALANCE.md に記載すること。
    - 係数などを用いる場合は、config.js に定義すること。
- `update(deltaTime)` 内でゲージの時間減衰を毎フレーム計算してください。
- **[重要]** 減衰量は一定ではなく、「基本減衰値 ＋（加速度係数 × (現在値 / 臨界値) の累乗）」という割合ベースの計算式を用いて、「臨界値（限界）に近づけば近づくほど、加速度的に強烈な減衰（復元力）が発生する」ロジックを実装してください。

### 2. デバッグ計器の拡張 (Visualizer.js)
- 第12層（DEBUG_OVERLAY）の描画処理に、PhaseManagerから取得した「現在ゲージ量（および割合%）」「直前獲得ポイント」「現在の減衰レート」の3項目をリアルタイム表示する処理を追記してください。
- 開発者がこの数値を見ながらテストプレイし、最適なバランス（マジックナンバー）をチューニングできるよう可視化することが目的です。

### 3. 大当り予兆エフェクトの実装
- ゲージの蓄積割合（0.0〜1.0）に応じて、ヘッダの中心から広がる無機質な「ソナー波紋」エフェクトを発生させてください。
- 割合が1.0に近づくほど、波紋の発生インターバルが短くなり、テンポと密度が増すように計算式を組んでください。

### 4. 発動タイムラインの実装 (PhaseManager.js 他)
ゲージが臨界値（MAX）に達した瞬間、以下のシーケンスを正確なタイムラインで実行してください。
1. **[突入]**: 状態を「ホワイト突入演出中 (`PHASE_WHITE_ENTER`)」へ移行し、物理演算を完全に停止（ステイシス）させる。同時に通常BGMにローパスフィルタをかけ、音をくぐもらせる。この間、タップ入力は受け付けないこと。
2. **[1秒経過後]**: 画面全体を覆い尽くす超巨大な「純白のフラッシュ（ホワイトアウト）」を描画する（第10層 GLOBAL_POST_EFFECT 等を活用）。
3. **[同時]**: 状態を「ホワイトステイシス中 (`PHASE_WHITE`)」へ移行し、ステイシス状態を解除する。同時に、既存のBGMを停止し、フィーバーBGMを「曲の頭（再生位置0）」から爆発的に再生スタートさせる。

## 【注意事項】
- ゲージの蓄積量や減衰係数などのマジックナンバーは、将来的な調整を見越して適切に定数化（設定ファイル等へ分離）してください。
- ホワイトアウトのフラッシュ描画は、既存のDOM要素操作は絶対に行わず、Canvasパイプラインのルールに従って実装してください。
- Step0で作成したガード条件（タップ不可等）が、演出中（`PHASE_WHITE_ENTER`）に正しく機能し、プレイヤーの誤操作によるバグを防いでいることを確認してください。
