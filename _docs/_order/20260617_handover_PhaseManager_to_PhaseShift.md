# 引き継ぎ資料：PhaseManager から PhaseShift 実装へ

この資料は、前回の開発セッション（v0.26.3）から次回の開発セッション（フェイズシフト実装 Step1）への文脈を引き継ぐためのドキュメントです。
新しいチャットセッションを開始した AI アシスタントは、まずこの資料を読み、現在のプロジェクト状態と次に実行すべきタスクを把握してください。

## 1. 現在のプロジェクト状態（v0.26.3 時点）

* **PhaseManager の導入完了:**
  システム・ロジック層 (`js/core/PhaseManager.js`) にステートマシンを導入し、ゲーム進行の状態を一元管理する構造が完成しています。
* **実装済みのフェイズ:**
  * `PHASE_START` (ゲーム開始待機中): パズル開始後1.5秒間。入力とLIFE減少をブロック。
  * `PHASE_NORMAL` (通常パズル時): 通常のゲームプレイ状態。
  * `PHASE_GAMEOVER` (ゲームオーバー演出中): LIFE0到達時。ステイシス（時間停止）を発動し、1.5秒間の余韻の後に `ResultScene` へ遷移する。
* **アーキテクチャの原則:**
  * **DOM操作は完全に禁止**されています（v0.12.2で廃止済み）。すべての描画は Canvas 上の Layer や Effect パイプラインを通して行われます。
  * `logic.js` の責務が整理され、ゲームオーバーの遷移ロジックはすべて `PhaseManager.js` 側に移管されています。

## 2. 次のセッションで実行すべきタスク

次に実行すべきタスクは、同ディレクトリ内にある指示書に従って進めてください。

**▶ 対象指示書:** `_docs/_order/20260616_order08_feat_PhaseShiftStep1_LogicAndEffects.md`

### 次のタスクの概要（PhaseShift Step1）
1. **ゲージロジックの実装 (`PhaseManager.js`):**
   * フェイズゲージ（初期MAX: 1000）の蓄積処理を追加（連鎖数や階層に応じたポイント加算）。
   * `update()` 内で、時間経過による「加速度的な減衰（復元力）」を計算するロジックを実装。
2. **デバッグ計器の追加 (`Visualizer.js`):**
   * 現在のゲージ量、割合、減衰レートなどを画面に描画し、バランス調整ができるようにする。
3. **予兆エフェクトの実装:**
   * ゲージの蓄積割合に応じた無機質な「ソナー波紋」を発生させる。
4. **発動タイムラインの実装:**
   * ゲージMAX時に `PHASE_WHITE_ENTER` へ移行。
   * 物理演算の完全停止（ステイシス）、BGMのローパスフィルタ化。
   * 1秒経過後に「純白のフラッシュ（ホワイトアウト）」を描画。
   * 同時に `PHASE_WHITE` へ移行し、ステイシス解除＆フィーバーBGMの頭出し再生を開始。

## 3. 注意事項
* 前回のバージョンは **v0.26.3** です。以降の作業でも **v0.26.X** のマイナーバージョン帯を維持し、「フェイズシフト実装」のシリーズとして管理してください。
* マジックナンバーは可能な限り `config.js` に定数として定義してください。
* 各ステップの完了時には、必ず `changelog.js` および `PROJECT_ARCHITECTURE.md` 等のバージョン表記を更新してください。
