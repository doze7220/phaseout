# 新色アンロック時の経験値効率（破壊数）補正の実装

## Goal
ロードマップ第2項目の「落下色数のレベル連動」により、Lv5以降で新色がアンロックされた際、新色の破壊数（`colorDestroyCounts`）が0スタートとなることで、全体の「最小破壊色カウント」が0に引き下げられ、獲得経験値効率が絶望的に低下してしまうバランス上の問題を回避します。
新色が追加されるタイミングで、既存のアクティブカラーの破壊数の平均値を算出し、それを新色の初期破壊数（補正値）として付与するロジックを実装します。

## 変更内容詳細

### 1. `js/core/StageManager.js` の改修
*   **`onLevelUp(newLevel)` メソッドの修正**:
    *   新色が `UNLOCKABLE_COLORS` から選出された際、それを `activeColors` に追加すると同時に以下の計算を行います。
    *   **既存色の破壊数平均の算出**: 現在の `GameState.activeColors` に含まれている各既存色の `GameState.colorDestroyCounts` の値を取得し、その平均値を計算します（小数点以下は `Math.floor` または `Math.round` で丸めてください）。
    *   **新色への補正値適用**: 新色の `GameState.colorDestroyCounts[newColorHex]` の初期値として、算出した平均値を代入します。（初期化時の `0` スタートを、この平均値スタートで上書きします）
    *   *(※これにより、新色が追加された瞬間でも、既存色と同等の破壊数を最初から持っている扱いになり、`logic.js` 側の経験値減衰ペナルティが誤爆するのを防ぎます。)*

### 2. バージョニングとドキュメント更新
*   `changelog.js`: 新しいバージョン（例: `v0.20.5`）として「機能追加: 新色アンロック時、既存色の破壊数の平均値を新色の初期破壊数として付与し、経験値効率の極端な低下を防ぐ補正ロジックを実装」を追記してください。
