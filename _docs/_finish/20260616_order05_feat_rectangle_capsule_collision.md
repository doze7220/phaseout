# 長方形ブロックのカプセル判定（連鎖判定緩和）実装指示書

## 1. 目的
現在の「中心点同士の距離計算」では、長方形の長辺方向の端に位置する宝石が、見た目上は近接していても連鎖範囲外と判定されてしまう問題が発生している。
厳密な物理衝突計算を導入して処理を重くするのではなく、「プレイヤーの面白さ（繋がりやすさ）」を最優先し、対象が長方形（Rectangle）の場合のみ「長辺に沿った内部の線分（芯）」からの最短距離を測る「カプセル距離判定」を導入して、判定の理不尽さを解消する。

## 2. 対象ファイル
*   `js/core/logic.js`

## 3. 修正内容詳細

### 【1】 線分と点の最短距離関数の追加
`logic.js` 内（モジュールレベル）に、軽量な数学処理として「線分と点の最短距離を計算するヘルパー関数」を追加してください。
*   ベクトルの内積と線分上の割合（0〜1へのクランプ処理）を用いて、点から線分への垂線、あるいは端点までの最短距離を算出する、一般的な軽量アルゴリズムを実装してください。
*   Matter.jsの重いCollision関数などは使わず、純粋なMath関数のみで構成してください。

### 【2】 areGemsTouching のロジック改修
`logic.js` の `areGemsTouching(g1, g2)` 関数を改修し、長方形に対するカプセル判定を組み込んでください。

1.  **既存定数の維持:** 
    判定の猶予には、既存の `config.js` からインポートしている `CONNECTION_THRESHOLD` （20px）をそのまま使用してください。
2.  **基本半径と形状の取得:**
    各Body（g1, g2）から、基本となる判定半径を取得してください。円の場合は `circleRadius`、長方形など多角形の場合は「幅と高さのうち、短い方の半分」を基本半径とします。
3.  **長方形（カプセル）の線分算出ロジック:**
    Bodyのカスタムプロパティ（`shape === 'rectangle'` など）を判定し、長方形の場合のみ以下の計算を行ってください。
    *   `position` と `angle` を使用して、中心から長辺方向へ伸びるベクトルを求めます。
    *   長辺の長さから両端の「基本半径」分を差し引いた、内部の「芯」となる線分の両端座標を算出してください。
4.  **距離の測定と判定:**
    *   **一方が長方形の場合:** 長方形側の「内部の線分」と、もう一方の「中心点」との最短距離を、先ほど作成したヘルパー関数で計算します。
    *   **両方が長方形の場合:** 処理の軽量化のため、厳密な線分同士の距離計算は避け、「g1の線分とg2の中心点」「g2の線分とg1の中心点」の2パターンを計算し、短い方の距離を採用してください。
    *   **どちらも長方形以外の場合:** 従来通り、中心点同士の直線距離を計算します。
5.  **最終評価:**
    算出した最短距離が `< (g1の基本半径 + g2の基本半径 + CONNECTION_THRESHOLD)` であれば `true` を返すようにしてください。

## 4. 確認事項
*   Matter.jsの `Collision.collides` 等の重い物理衝突計算は絶対に使用しないこと（意図的に設定されているガバガバ判定・猶予を消さないため）。
*   追加する計算は純粋なMath計算のみとし、大量の宝石が連鎖判定されてもFPSが低下しないよう軽量に保つこと。
*   長方形が縦や斜めに立っている状態でも、長辺の端付近から隣接する宝石へ確実にレーザーが伝播するようになっていること。
