あなたは優秀なゲームエンジニアです。
パズルゲーム『Phase Out: Cluster Stirring』において、UI改修のStep1bとして、「プレイタイマー」および「秒間LIFE減少量（レート）」のリアルタイム表示をヘッダー左上に実装し、それに伴う「ヘッダーレイアウトの排他制御（スコア領域の調整）」を行います。
`PROJECT_ARCHITECTURE.md`のルールに従い、ロジックとDOM更新の責務を分離して実装してください。

【対象ファイル】
・index.html (ヘッダー左上へのDOM追加、構造整理)
・style.css (ヘッダーレイアウトのFlexbox調整、タイマー用スタイル)
・logic.js (秒間LIFE減少量を算出する関数のエクスポート)
・ScreenEffects.js (毎フレームのテキスト更新処理)

【実装要件】
1. ヘッダーレイアウトの再構築と排他制御 (`index.html`, `style.css`)
・既存のスコア表示が長く（インフレして）なった際にタイマー表示と重ならないよう、ヘッダー内のレイアウトをFlexbox等で明確に左右に分割してください。
・左側（新設）: タイマーおよび減少レートを表示するコンテナ（例: `flex: 0 0 auto; width: 35%;` 等で幅を固定・確保）。
・右側（既存）: スコアおよび単位を表示するコンテナ。左側のタイマーエリアに侵食しないよう、`max-width` や `flex: 1 1 auto;` などを設定して表示領域を制限（狭める）してください。
・スコアコンテナ幅が制限された状態でもレイアウトが崩れないよう、文字サイズを少し小さく調整するか、`white-space: nowrap;` 等を適用して綺麗に右寄せで収まるようにスタイリングしてください。

2. タイマー・減少レートのDOM要素追加 (`index.html`, `style.css`)
・左側の新設コンテナ内に、上段にタイマー（例: `<div id="play-timer">00:00:00</div>`）、下段に秒間減少レート（例: `<div id="life-decay-rate">- 0 /s</div>`）を配置してください。
・タイマーはデジタル風の等幅フォント（monospace等）にし、減少レートのテキストカラーは、ダメージを連想させるオレンジ系（#FF9500等）にしてください。

3. LIFE減少レート取得ロジックの公開 (`logic.js`)
・現在実装されている時間経過によるLIFE減少処理に用いられている計算式を元に、**「現在の1秒あたりのLIFE減少量（最終値）」**を算出する関数（例: `getCurrentLifeDecayRate()`）を作成し、外部にエクスポートしてください。
・この計算には、「経過時間による増加」と「GameState.levelによる指数関数的な増加」のすべての要素を加味し、秒間ダメージとしての数値を返してください。

4. リアルタイムUI更新 (`ScreenEffects.js`)
・`requestAnimationFrame` 等の描画・更新ループ（GaugeManagerのupdate内など、毎フレーム呼ばれる箇所）にて、以下のDOM更新を行ってください。
　1. **タイマー:** `GameState.playStartTime` からの経過時間を算出し、「分:秒:ミリ秒（10ミリ秒単位2桁）」（例: `01:23:45`）の形式で `#play-timer` に表示してください。
　2. **減少レート:** `logic.js` の `getCurrentLifeDecayRate()` を呼び出して現在の秒間減少量を取得し、「`- 〇〇 /s`」（小数点は丸めるか1桁程度にフォーマット）の形式で `#life-decay-rate` に表示してください。

【遵守事項】
・DOMアクセス（`innerHTML`や`textContent`の書き換え）は最低限の要素に対してのみ行い、レイアウトの再計算（リフロー）による負荷が高まらないように注意してください。
・ロジック計算は必ず `logic.js` 側で行い、`ScreenEffects.js` は値を受け取って表示するだけの責務を守ってください。
