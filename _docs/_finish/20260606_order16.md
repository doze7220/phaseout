あなたは優秀なゲームエンジニアです。
パズルゲーム『Phase Out: Cluster Stirring』において、スマートフォンでの描画負荷（FPS低下）を改善するため、現在の各種エフェクトを3段階（FULL / LITE / NONE）で一括制御する「軽量化オプション」を実装します。

現在のアーキテクチャルール（`PROJECT_ARCHITECTURE.md` 等）を厳守し、以下の要件を満たしてください。

### 【実装要件】
#### 1. 設定とUIの統合
*   **config.js** : `AppConfig.EFFECT_LEVEL` (FULL/LITE/NONE) を新規追加します。既存の `VISUALIZER_MODE` は廃止するか、内部ロジックで `EFFECT_LEVEL` に完全に依存して切り替わるように統合します。
*   **index.html (UI追加)** :
    *   **タイトル画面:** 左下の「GEM表示設定トグル（GEM : [FLAT] [RICH]）」の**すぐ上**に、エフェクト切り替え用のUI「EFFECT: [FULL] [LITE] [NONE]」を追加してください。
    *   **コンフィグ画面:** モーダル内の既存の「ビジュアライザ」トグルUIを「エフェクト設定（FULL / LITE / NONE）」にリネーム・拡張し、タイトル画面のUIと状態を常に同期させます。
*   **main.js** : トグルUIのイベントリスナーを実装し、切り替え時に `AppConfig.EFFECT_LEVEL` を更新すると同時に、連携するエフェクトのモード変更を各マネージャーへ通知します。

#### 2. 初期値の自動判定と設定の永続化 (localStorage)
*   **デバイス自動判定:** `config.js` または初期化処理において、実行環境がモバイル端末（UserAgentやタッチ対応等で判定）であればデフォルトを `LITE`、PCであれば `FULL` に設定してください。
*   **localStorageへの保存:** ユーザーがUIから設定を変更した際は、その値を `localStorage` に保存してください。
*   **読み込み優先度:** アプリ起動時は、デバイスの自動判定よりも `localStorage` の保存値を優先して読み込んでください。

#### 3. エフェクトレベルの定義と制御内容
**【FULL（フル演出・高負荷）】** 現在の仕様そのまま。
*   レーザー：加算合成 (`lighter`)
*   パーティクル：全種（破片、火花、バースト）最大数
*   波紋、画面揺れ、宝石フラッシュ、SVGゲージ：すべて有効
*   ビジュアライザ：WAVEモード（リッチな波形表現）

**【LITE（ゲーム性維持・中負荷）】** DOM生成・加算合成をカット
*   レーザー：加算合成を `source-over`（通常描画）等に変更
*   パーティクル：バースト火花をカット、破片は50%に間引き
*   波紋：DOMの動的生成とCSSアニメーションをカット（OFF）
*   画面揺れ、宝石フラッシュ：維持
*   SVGゲージ：高負荷なCSSフィルタ（drop-shadow）をカットし、ストローク幅・色の変化のみで表現
*   ビジュアライザ：BLOCKモード（軽量なスリットメーター表現）

**【NONE（超軽量・低負荷）】**
*   レーザー：アルファ値の低い極細い直線1本のみ、またはOFF
*   パーティクル：一切発生させない
*   波紋、画面揺れ：OFF
*   宝石フラッシュ：重ね描画をカットし、単なる沈み込み表現のみ
*   SVGゲージ：フラッシュ演出を完全カット
*   ビジュアライザ：BLOCKモードをベースに、時間経過のランダムな揺らぎ（脈動）アニメーションをカットし、プレイヤーが宝石を破壊したタイミングでのみ情報を更新（超軽量版）

#### 4. 各ファイルへの実装指示・アーキテクチャ厳守事項
*   **main.js**: `createRipple` 関数の**冒頭**で `AppConfig.EFFECT_LEVEL` が `FULL` 以外なら即座に `return` する処理を追加してください（ScreenEffects.jsではなくmain.jsで行うこと）。
*   **effects.js (Facade)**: `spawnParticles`等の呼び出し前にレベルを判定し、マネージャーへ渡す前に生成数の調整・スキップを実施してください。
*   **LaserEffect.js**: LITE/NONE時の加算合成の無効化を実装する際、**既存の `ctx.save()` と `ctx.restore()` のブロック構造は絶対に破壊せず、コンテキスト状態の保護ルールを厳守**してください。
*   **ScreenEffects.js**: SVGゲージフラッシュ用CSSクラス（heal-flash, damage-flash）の付与を LITE/NONE で抑制してください。
*   **renderer.js**: NONE時のスタンプ重ね描画のスキップ等を実装してください。
