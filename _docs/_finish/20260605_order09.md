あなたは優秀なゲームエンジニアです。
パズルゲーム『Phase Out: Cluster Stirring』において、今後の「インフレするスコアと経験値ロジック」を視覚的に確認できるよう、UI（ゲージ）の改良と追加を行います。
アーキテクチャの「DOM/CSSバインディング規則」に従い、以下の要件を実装してください。

【対象ファイル】
・index.html / style.css (UI要素の追加とスタイリング)
・config.js (GameStateへのEXP関連変数の追加)
・ScreenEffects.js / logic.js (ゲージのアニメーションと状態反映)

【実装の要件】
1. EXP（経験値）ゲージの新設と、撹拌Lv表示の「重ね合わせ」
・中段パズルエリア上部に、次レベルまでの進捗を示す「EXPゲージ（プログレスバー）」をHTML/CSSで追加してください。
・現在独立して表示されている「撹拌Lv」のテキスト要素を、新設したEXPゲージ（またはLIFEゲージの頂点）と重なるように絶対配置（`position: absolute` と `z-index`）を用いてレイアウトし、UIとして一体化させてください。文字がゲージの背景に埋もれないよう、テキストにドロップシャドウや縁取り（`text-shadow`）をつけて視認性を確保してください。
・★重要：このUI要素がパズルエリアとヘッダーの境界を跨ぐ（はみ出す）レイアウトになることを想定し、親コンテナが `overflow: hidden` で要素を見切れさせないよう、適宜 `overflow: visible` やZ-Indexの設定を行い、画面上で美しくオーバーレイ表示されるよう調整してください。
・GameState に `exp` と `nextLevelExp` を追加し、毎フレームの描画ループでEXPゲージの幅（%）とレベルテキストが追従するようにバインディングしてください。

2. LIFEオーバーフロー（超過）演出の土台構築
・連鎖によるLIFE回復処理（logic.js）において、回復量が `maxLife` を超えた場合、その「超過分（オーバーフロー）」を検知する処理を入れてください。
・超過が発生した際、ScreenEffects.js を呼び出して、新設した「EXPゲージ」が白くフラッシュする、あるいはLIFEゲージからEXPゲージへ光が飛ぶような「超過分が経験値に変換されたことが直感的にわかる演出」を実装してください。（現時点ではダミーの経験値を加算してテストできるようにしてください）

3. ゲージ表現の強化
・LIFEゲージおよびEXPゲージが大きく変動した際、CSSの `transform: scale()` や `box-shadow` などを一時的に変化させ、「エネルギーが注入された（または激しく削られた）」ことが分かりやすくなるキレのあるアニメーション（CSSクラスの付与・削除など）を追加してください。

【遵守事項】
・描画層の分離: ゲージの操作は `ScreenEffects.js` や専用のUI更新関数に集約し、Matter.jsの物理ループ内（physics.js）で直接DOMを操作しないこと。
・既存のLIFEゲージ（SVGベースのアニメーション）の仕組みと競合しないよう注意すること。
