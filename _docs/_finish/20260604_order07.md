※既存の物理挙動やパズルロジックは変更せず、UIの追加・調整のみを行ってください。

##### 【1. 上段UI（ui-header）の構築（index.html / style.css）】
・`index.html` の上段エリア内に、以下の要素（ダミーまたは実数値と紐付け）を配置してください。
  - **左上**: オプション/ポーズボタンのダミーアイコン（例：「⚙」など）。
  - **中央**: 現在の「撹拌Lv（Lv.1など）」と、「スコア（例：1.000万）」の表示。既存のUI要素をここに移動・統合してください。
  - **右上**: 現在の「セブンスパレット（陣営）」を示すダミーのエンブレムアイコン枠。
・`style.css` にて、Flexbox等を用いてこれらの要素がバランスよく横並びになるようスタイリングしてください。背景は少し暗めの半透明（例: `rgba(0,0,0, 0.6)`）などにし、視認性を高めてください。

##### 【2. 下段UI（ui-footer）のダミー構築（index.html / style.css）】
・`index.html` の下段エリア内に、今後のキャラクター実装を見据えた以下のダミーUIを配置してください。
  - **左側**: キャラクター（ステラ）の顔アイコンを表示するための四角または円形のプレースホルダー画像（ダミー枠）。
  - **右側**: 「アクティブスキル」や「プリズムリンク」の発動を示すための、2つのダミーボタン枠（ゲージ風の装飾を添える）。
・`style.css` にて、ゲームらしいサイバー調、あるいはガラス調のボーダーなどを設けてスタイリングし、パズル中段の邪魔にならないデザインにしてください。

##### 【3. LIFEゲージ（枠線）の中段エリアへの完全連動（effects.js / logic.js）】
・現在のLIFEゲージ（SVG線）を、画面全体ではなく**中段（puzzle-area）の境界線に正確に沿う**ように改修します。
・`config.js` で定義された `LAYOUT_CONFIG`（`APP_WIDTH` と `APP_HEIGHT - HEADER_HEIGHT - FOOTER_HEIGHT`）を用いて、中段エリアの幅と高さをJavaScriptで取得してください。
・取得した幅と高さを元に、ゲージ用SVGの `width`, `height`, および `viewBox` を動的に設定してください。
・SVGの `stroke-dasharray`（外周の長さ）および `stroke-dashoffset` の計算ロジックを、この「動的に取得した中段の幅・高さ」を用いた計算式（例: `周長 = (幅 + 高さ) * 2`）に修正し、LIFEのパーセンテージに応じて正確に枠線が削れていくよう連動させてください。
