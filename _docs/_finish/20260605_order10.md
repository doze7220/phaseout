`PROJECT_ARCHITECTURE.md`と`PROJECT_FUNCTION_INDEX.md`のルールに従い、DOM/SVG操作とロジック計算の責務を厳密に分離して実装してください。

【対象ファイル】
・config.js (表示用GameState変数の追加)
・logic.js (内部経験値の即時計算、旧オーラ演出の削除)
・ScreenEffects.js (GaugeManagerの改修、連続レベルアップ演出、旧オーラ演出の削除)

【実装要件】
1. 既存のパネル（オーラ）エフェクトの完全削除（クリーンアップ）
・過去に実装された「一番多く破壊している宝石の色でヘッダーパネルがフラッシュする」演出を完全に削除してください。
・`style.css`、`logic.js`、`ScreenEffects.js` 等に存在する `--aura-color` 更新ロジックおよび `box-shadow` の操作をパージしてください。

2. Lv表記デザインの刷新とスリム化 (`index.html`, `style.css`)
・現在、中段パズルエリア上部に絶対配置されている「撹拌Lv（Level）」の表記UIについて、内部に存在している不要な「プログレスバー（背景ゲージ等）」の要素をHTMLおよびCSSから完全に削除してください。
・Lv表記のコンテナの横サイズ（width/padding）を小さくまとめ、コンパクトな表示にしてください。
・コンテナの形状を「両端が楕円（カプセル型）」から、「丸角の四角（例: `border-radius: 6px` 程度）」へとシャープでシステムチックなデザインに変更してください。

3. 状態定義の追加 (`config.js`)
・`GameState` にアニメーション追従用の `displayExp: 0` と `displayLevel: 1` を追加してください。

4. レベル・経験値の内部即時計算 (`logic.js`)
・経験値（EXP）を獲得した際、アニメーションを待たずに内部の `GameState.exp` と `GameState.level` を即座に計算・確定させてください。
・一度に複数のレベルが上がる場合も想定し、`while`ループ等を用いて `exp` が `nextLevelExp` を上回っている限り `level` を上げ、内部状態を常に最新にしてください。（※経験値の計算式自体は今回は変更しません）

5. LIFEゲージとEXPゲージの同心矩形化 (`ScreenEffects.js` - `GaugeManager`)
・現在の中段パズルエリアを囲むSVGのLIFEゲージの「内側」に、全く同じ形状の「EXPゲージ（白または虹色の線）」のパスを追加してください。
・EXPゲージは `(GameState.displayExp / 内部のnextLevelExp)` の割合で下から上へと満ちるように `stroke-dashoffset` を更新してください。

6. 連続レベルアップ演出（アニメーションキュー処理）
・`ScreenEffects.js` の `GaugeManager#update` または描画ループにて、`GameState.displayExp` を実際の `GameState.exp` に向けて増加させてください（約0.5秒でゲージが空からMAXになる猛烈な速度）。
・`displayExp` がその `displayLevel` における `nextLevelExp` 以上になった瞬間、以下の処理を実行します。
　1. `effects.js` 経由でレベルアップのポップアップ演出を表示する。
　2. EXPゲージ全体が一瞬フラッシュするような演出を入れる（CSSクラスの付与等）。
　3. `displayExp` から `nextLevelExp` を減算し、`displayLevel` を +1 する。
　4. 実際の `exp` に追いつくまで、何度も連続してゲージが満ちてはレベルアップが弾けるようにする。

【遵守事項】
・UI（画面表示）のアニメーションと、内部のロジック（GameStateの確定）が分離されていることを意識してください。
