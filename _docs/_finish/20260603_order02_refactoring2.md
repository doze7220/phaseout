あなたは優秀なゲームエンジニアです。
現在作成中の「物理演算ベースの連鎖パズルゲーム」について、肥大化したコードを整理する【リファクタリング（SPA基盤構築）】と、パフォーマンスを落とさずに視覚的没入感を高める【宝石のカスタム描画（キャンバスキャッシュ最適化）】を同時に実行してください。
※既存の機能、物理挙動の手触り、エフェクトのタイミングは一切変更しないこと。

### 【1. モジュール分割とファイル構成（ES Modules）】
現在のコードを以下の機能ごとに分割し、`type="module"` で読み込む構成にしてください。
- `index.html` (エントリーポイント、シーンコンテナ定義)
- `style.css` (デザイン、シーンの表示/非表示制御)
- `main.js` (全体の初期化、シーン呼び出し)
- `config.js` (すべてのパラメータ、基本色、物理設定の定数化)
- `physics.js` ( Matter.jsエンジンの初期化、壁生成、宝石スポーン)
- `logic.js` (タップ入力、許容距離ベースのBFS連鎖探索、破壊・ペナルティ判定)
- `score.js` (巨大数計算、漢字単位のループ、小数点以下3桁固定フォーマット)
- `effects.js` (レーザー伝播、パーティクル演出、画面揺れ)
- `scene.js` (SPA的なシーン遷移管理)
- `renderer.js` (★新規：カスタム描画とキャッシュ管理)

### 【2. シーン管理とリザルト（SPA化）】
- `index.html` にコンテナを作成（`scene-title`, `scene-home`, `scene-puzzle`, `scene-gacha` 等）し、`scene.js` で表示を切り替えてください。パズル以外の画面はモック（仮ボタンとテキスト）で構いません。
- リザルト画面は、パズル終了時に盤面を背景に残したまま、`scene-puzzle` 内のオーバーレイ要素としてフェードイン表示させてください。

### 【3. 宝石のカスタム描画とキャッシュ最適化（renderer.js）】
Matter.jsの物理衝突判定（Collision形状）はそのまま維持し、見た目の描画処理のみをリッチかつ軽量に書き換えます。

1. 【キャンバスキャッシュ生成（プレレンダリング）】
   - ゲーム初期化時に、7色×各形状（丸、長方形、正方形、正三角形など）のバリエーションごとに「オフスクリーンキャンバス（未描画Canvas要素）」を作成し、一度だけ描画してメモリにキャッシュしてください。
   - 描画デザイン（ダイヤモンドカット風）：
     - 角に少し丸みを持たせたベース図形に、属性に応じた色（グラデーション推奨）を塗る。
     - 図形の内側に、半透明の白（`rgba(255,255,255,0.5)`等）でファセットや放射状の線を描き、立体的な「カット面」を表現。
     - 光源からのツヤ感を出すため、左上に明るいハイライト（円弧やポリゴン）をオーバーレイ。

2. 【Matter.jsの描画フックとスタンプ描画】
   - 宝石のデフォルト描画は無効化（`render.visible = false` 等）してください。
   - `Events.on(render, 'afterRender', ...)` フックを用い、各宝石Bodyの `position` と `angle` に合わせてCanvasコンテキストを `translate` / `rotate` してください。
   - 複雑な描画命令（`lineTo`等）は行わず、先ほど生成したキャッシュ画像を `drawImage()` でスタンプするように高速描画してください。

### 【4. 実行時の注意事項】
- リファクタリング後もブラウザ上でエラーなく動作すること。
- 将来的に `renderer.js` のキャッシュ生成処理を「本番用画像データの `loadImage()`」に差し替えるだけで済むように、クラスや関数のインターフェース（Facadeパターンのような形）を綺麗にカプセル化しておくこと。
