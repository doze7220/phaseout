# 軽量化（エフェクト）オプション導入 仕様書 (Order 16)

パズル画面の描画負荷（特にスマートフォンでのFPS低下）を改善するため、現在の各種エフェクトを3段階（FULL / LITE / NONE）で一括制御する仕組みを導入します。

## 1. 設定とUIの統合

*   **`config.js`**: `AppConfig.EFFECT_LEVEL` (FULL/LITE/NONE) を新規追加します。既存の `VISUALIZER_MODE` は廃止するか、内部ロジックで `EFFECT_LEVEL` に完全に依存して切り替わるように統合します。
*   **`index.html`**: コンフィグモーダル内の「ビジュアライザ」トグルUIを「エフェクト設定（FULL / LITE / NONE）」にリネーム・拡張し、この設定一箇所で全体を管理できるようにします。
*   **`main.js`**: 
    * トグルUIのイベントリスナーを修正し、切り替え時に `AppConfig.EFFECT_LEVEL` を更新すると同時に、連携するエフェクトのモード変更を各マネージャーへ通知します。
    * `createRipple` 関数の冒頭で `AppConfig.EFFECT_LEVEL` が FULL 以外なら即座に return する処理を追加します。

## 2. エフェクトレベルの定義と制御内容

### 2.1. FULL（フル演出・高負荷）
現在の仕様そのまま。すべての演出が有効になります。
*   **レーザー**: 加算合成 (`lighter`) を使用し、全階層・全発光を描画。
*   **パーティクル**: 破片（Particles）、火花（Sparks）、バースト（BurstSparks）のすべてを最大数発生。
*   **波紋（タップ時）**: CSS+DOMアニメーションで発生。
*   **画面揺れ**: 連鎖終了時に発生。
*   **宝石フラッシュ**: 連鎖時に重ね掛け描画による発光と沈み込みを表現。
*   **SVGゲージ**: `drop-shadow` フィルタ等を用いたリッチなフラッシュ（明滅）アニメーション。
*   **ビジュアライザ**: WAVEモード（リッチな波形表現）。

### 2.2. LITE（ゲーム性維持・中負荷）
ゲームの爽快感（何が起きているかの視認性）を保ちつつ、特に GPU・CPU 負荷の高い合成や DOM の生成をカットします。
*   **レーザー**: 加算合成（`lighter`）を `source-over`（通常描画）に切り替え、光の滲み（シャドウ）計算を省略して直線を高速描画。
*   **パーティクル**: バースト火花などの細かいパーティクル生成をカットし、破片（Particles）のみ発生数を 50% に間引いて描画。
*   **波紋（タップ時）**: DOMの動的生成とCSSアニメーションはリフロー負荷が高いためカット（OFF）。
*   **画面揺れ**: 発生する（維持）。
*   **宝石フラッシュ**: 維持。
*   **SVGゲージ**: 高負荷な SVG フィルタ (`filter: drop-shadow()`) の CSS クラス付与をカットし、ストローク幅と色の変化のみで表現。
*   **ビジュアライザ**: BLOCKモード（軽量なスリットメーター表現）。

### 2.3. NONE（超軽量・低負荷）
物理演算と宝石の描画のみに絞り、フレームレートの維持を最優先します。
*   **レーザー**: 描画を完全にスキップするか、アルファ値の低い極めて細い直線1本のみを描画。
*   **パーティクル**: 一切発生させない。`ParticleManager` 内のアップデート処理をスルー。
*   **波紋（タップ時）**: OFF。
*   **画面揺れ**: OFF。
*   **宝石フラッシュ**: 重ね掛けの描画処理をカットし、単に宝石のサイズが一時的に小さくなる（沈み込み）だけの表現に変更。
*   **SVGゲージ**: フラッシュ演出を完全にカット。
*   **ビジュアライザ**: BLOCKモードをベースとした超軽量版。プレイヤーが宝石を破壊したタイミングでのみ情報を更新し、時間経過によるランダムな揺らぎ（脈動）アニメーションをカットすることで再描画負荷を極限まで抑えます。

## 3. 各マネージャーへの分岐ロジック実装

*   **`effects.js` (Facade)**:
    *   `spawnParticles`, `spawnSparks`, `spawnBurstSparks` の呼び出し前に `AppConfig.EFFECT_LEVEL` を判定し、生成数の調整やカットを実施。
    *   `triggerScreenShake` などで NONE の場合は即時 Return する処理を追加。
*   **`LaserEffect.js`**:
    *   `ctx.globalCompositeOperation` の切り替え（LITEやNONEでは `lighter` を使わず `source-over` 等に変更）。※この際、既存の `ctx.save()` と `ctx.restore()` のブロック構造は絶対に破壊せず、コンテキスト状態の保護ルールを厳守します。
*   **`ScreenEffects.js`**:
    *   SVG ゲージフラッシュ用クラスの付与（`heal-flash`, `damage-flash`）を LITE/NONE で抑制。
*   **`renderer.js`**:
    *   `drawRichGem` 等のスタンプ描画・フラッシュ（`tapEffectTimer`）の処理を NONE 時にスキップ。
