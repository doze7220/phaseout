## AI駆動開発向け マスタードキュメント（プロジェクト憲法）



本ドキュメントは、AI-IDE（Cursor, Copilot, Claude等）が本プロジェクトのコードを生成・修正する際の「絶対の法律（憲法）」である。

AIはコード生成前に必ず本ドキュメントを参照し、以下のルールと手順に違反する実装を**固く禁ずる**。

この文書の最大の目的は、AIに「必要なファイルだけを解析させ、余計な場所を触らせないこと」である。

### 🚫 2. 絶対禁止事項（アーキテクチャ維持のためのルール）



*   `Scene` に物理演算・衝突判定・生成ロジックを書いてはならない。

*   `Scene` に描画処理（Canvas API等）を書いてはならない。

*   `Entity` から直接 `Renderer` を呼んではならない（描画の分離）。

*   `Entity` 同士が直接参照してはならない（必ずSystemかEventBusを経由すること）。

*   `Manager` 同士の直接参照による循環依存を禁止する。

*   `EventBus` を経由せずに、他クラスの副作用を直接発生させてはならない。

*   `RenderManager` にゲームロジック（座標移動や当たり判定）を書いてはならない。

*   **【JSDoc記述の義務】 JSDocコメント（`/** ... */`）を記述せずに、クラス、メソッド、プロパティを新規作成・修正することを固く禁ずる。**

*   **【Manager巨大化禁止】** クラスの行数が500〜1000行を超え始めた場合、必ず新しい `System` へ分離すること。



---



### 📋 3. 責務一覧表（AIはこれに従い処理を配置すること）



| モジュール層 | ⭕ 許可されている責務 | ❌ 禁止されている責務 |

| :--- | :--- | :--- |

| **Entity** | 状態（座標、HPなど）の保持、update() | draw()の実行、副作用の直接発火、Scene遷移 |

| **Renderer** | draw()による画面への描画 | 物理演算、AI思考、Event発行 |

| **Scene** | 各Managerの初期化、update/renderの呼び出し、シーン遷移 | 当たり判定、描画ロジック、AIロジック |

| **System** | 専門処理、**必要最小限の内部状態保持**（Cameraのtarget等） | ゲーム全体の永続状態保持、巨大な状態の集中管理 |

| **Data** | JSON等による静的なパラメータ定義 | 動的な状態の保持、メソッドの実行 |

| **Core** | インフラ機能の提供（Asset, Time, Event, Storage等） | 特定のゲームロジック、描画処理 |



---



### ➡️ 4. 依存方向（Importの許諾ルール）



**【許可される依存方向】**

*   `Scene` ➡️ `Systems` ➡️ `Entities` ➡️ `Data`

*   `Renderers` ⬅️ `CameraSystem`

*   **各層からCore層への依存**

    *   `Scene` ➡️ `Core`

    *   `Systems` ➡️ `Core`

    *   `Entities` ➡️ `Core`（最小限に留めること）

    *   `Renderers` ➡️ `Core`



**【❌ 絶対禁止の依存方向（逆流）】**

*   ❌ `Data` ➡️ `Entity`

*   ❌ `Entity` ➡️ `Renderer`

*   ❌ `Renderer` ➡️ `Entity`

*   ❌ `System` ➡️ `Scene`

*   ❌ **`Core` ➡️ `Scene` / `Systems` / `Entities` / `Renderers`**（Coreからの逆流禁止）

### 🤖 5. AI作業手順（厳守事項）



新機能追加やバグ修正時、AIは必ず以下の順序で作業を実行すること。



1. **本憲法（`PROJECT_CONSTITUTION.md`）を読む。**

2. 修正対象となるフォルダ・システムを特定する。

3. **必要なファイルのみを読み込む。**

4. **関係ないフォルダは絶対に解析・修正しない（クォータ節約）。**

5. 修正後は必ずJSDocを更新・追加する。

6. 新しい責務違反や循環参照が発生していないか確認する。

7. 既存クラスが肥大化しそうな場合は、新しい `System` を作成する。



---



### 📂 6. フォルダ責務ガイド（AI向けクォータ節約フロー）



*   **Configs:** 画面サイズ、FPS、パラメータ等の定数変更時に参照。ロジック変更時は参照禁止。

*   **Core:** Asset/Storage/Input基盤、EventBus、TimeManager変更時に参照。ゲームロジック変更時は参照禁止。

*   **Systems:** 衝突、AI、エフェクト、カメラ等の専門処理追加時に参照。描画・UI変更時は参照禁止。

*   **Renderers:** 描画レイヤー追加、UI、デバッグ描画等の変更時に参照。物理・状態変更時は参照禁止。

*   **Entities:** キャラクターや弾の物理挙動、HP等の状態追加時に参照。描画・AI思考変更時は参照禁止。

*   **Data:** 機体・武器データ追加、JSON調整時に参照。ロジック・描画変更時は参照禁止。

*   **Scenes:** シーン遷移、シーン初期化処理の追加時に参照。物理演算・描画ロジック変更時は参照禁止。

*   **Utils:** 共通関数、デバッグ機能追加時に参照。



---



### ⏱️ 7. GameLoopの絶対順序（UpdateManager管轄）



1. `TimeManager.update()` (deltaTime取得)

2. `InputManager.update()` (座標補正含む入力取得)

3. `Scene.update()` 呼び出し（`AISystem.update()` ➡️ `Entities.update()`）

4. `CollisionSystem.check()` (確定座標での衝突判定)

5. `EventBus.process()` (キューに溜まったイベントの一括処理)

6. `EntityManager.cleanup()` (死亡Entityの回収)

7. `CameraSystem.update()` (全処理後のカメラ移動)

8. `RenderManager.draw()` (全Rendererへの描画指示)
