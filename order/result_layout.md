# リザルト画面 (Result) レイアウト・アニメーション仕様

## 1. 画面構成とレイヤー
- **画面遷移**: `SceneManager.pushScene` を使用したモーダル表示。パズル画面の描画を裏に残したまま、その上に暗幕（`rgba(0, 0, 0, 0.8)`）を重ねて表示する。
- **描画レイヤー**: 第9層 (`MODAL_UI` レイヤー) の単一キャンバスで描画を完結させる。DOM要素やScrollAreaへの依存は存在しない。
- **UIコンセプト**: 「1ページ構成のサイバー風システムコンソールUI（HUDレイアウト）」。2ページ構成を廃止し、すべてのリザルト情報を1画面に集約する。

## 2. 表示要素一覧とレイアウト
ベース解像度の中央X座標を `centerX` (width / 2) とする。

### 2.1 全体装飾（サイバーフレーム）
- **外枠**: 画面全体を囲うシアン色の細い矩形線（`rgba(0, 255, 255, 0.3)`）。
- **タイトル**: 上部中央に `[ RESULT ]` を表示（シアン色、ボールド16px）。
- **コーナーブラケット**: 画面の四隅に配置されるL字型の装飾線（一辺20px）。

### 2.2 上段：巨大スコアの壁 (Huge Score Wall)
最終獲得スコアを巨大なスプライト文字で描画する領域。
- **20桁改行ロジック**: スコアが長大な場合、下（1の位）から数えて20桁（※漢字単位を除く純粋な数字文字数）ごとに自動で改行される。
- **視覚的センタリング**: 「単位（万・億など）を除いた、純粋な数字20文字分の幅」の中心が `centerX` とピタリと重なるように描画開始座標を逆算する。
- **行の右揃え**: 複数行にまたがる場合、最も長い行の右端を基準として、すべての行が右寄せ（Right-Align）で配置される。

### 2.3 中段：詳細サマリー (Summary)
プレイ実績を簡潔に表示する領域。ラベルと値の境界（コロン `:`）が `centerX` に位置するよう整列される。
- **配置要素**:
  - `LEVEL : {level}`
  - `TIME : {mm:ss}`
  - `1 TAP MAX SCORE : {スコアスプライト}`
- **発光アイコン**: `1 TAP MAX SCORE` のラベルの左横に、最大スコアを記録した際のベース色（`maxScoreColor`）で発光する円形アイコン（ドロップシャドウ付き）を描画する。

### 2.4 下段：戦果分析テーブル (Stats Table)
各色ごとの消去数や累計スコアのブレークダウンを表示する領域。
- **ヘッダー**: `DISRUPT` (消去数)、`TOTAL SCORE` (累計スコア)、`SKILL` (スキルボーナス) の3列をシアン色で表示。下部にシアンの区切り線を引く。
- **TOTAL行 (1行目)**: すべての色の合算実績を表示。フォントはボールドで白色。
- **色別行 (2行目以降)**: `GameState.totalScorePerColor` 等に基づき、実際にプレイ中に消去した色のみを表示する。
- **色アイコン**: 各行の左端に、その色で発光する円形アイコンを描画する。
- **背景ブロックメーター**: 各色の `TOTAL SCORE` の値に応じて、テーブル行の背景に白の半透明（`rgba(255,255,255,0.05)`）の横長ブロックが右端から左へ伸びるインジケーターを描画する。全体スコアに対する割合で長さが決定される。
- **スコアフォーマット**: テーブル内に収めるため、スコアは7桁以上の場合「X.YZ万/億/兆...」のような短縮フォーマット（`formatShortScore`）で表示される。

## 3. アニメーションと遷移仕様

### 3.1 アニメーション・フェーズ
リザルト画面は以下のフェーズで進行する。
1. **INIT**: 初期化。設定によりアニメーションの有無を判定。
2. **DRUMROLL (0s ~ 2.0s)**:
   - 全ての数値（巨大スコア、Max Score、テーブル内の各数値）が、`0` から確定値に向かってカウントアップするドラムロール演出。
   - 進行度は Ease-Out Quad 関数で補間される。
   - 要素ごとにフェードインの遅延（上段: 0ms, 中段: 200ms, 下段: 400ms）が設定されている。
   - この間、SE（システム音）は無音とする。
3. **WAIT (完了直後 ~ +0.8s)**:
   - ドラムロールが完了（実値到達）した瞬間に、画面全体を揺らすエフェクト（`ScreenEffects.triggerScreenShake`）と、重めの衝撃音（`SoundManager.playSE('BREAK_BURST')` 等）が1度だけ発火する。
   - 完了後 0.8秒間 は、タップ入力による次画面への遷移がブロックされる（誤操作防止ウェイト）。
4. **DONE (0.8s以降)**:
   - 画面下部に `[ TAP TO TITLE ]` が点滅表示（周期0.5秒）される。
   - 画面のどこをタップしてもタイトル画面へ遷移（`SceneManager.changeScene(new TitleScene())`）する。

### 3.2 スキップ機能とコンフィグ設定
- **手動スキップ**: `DRUMROLL` 中に画面をタップすると、アニメーションを即座にスキップし、全要素が確定した状態（`DONE`・ウェイト開始時点）へ飛ぶ。この際にも完了時の画面揺れと衝撃音は再生される。
- **設定によるOFF**: `AppConfig.RESULT_ANIMATION` が `false` の場合、フェードインやドラムロールは完全にカットされる。画面表示から `0.0秒` の時点で全要素がカンスト表示され、衝撃音と画面揺れが一度だけ発火した後、即座に0.8秒の誤操作防止ウェイト状態へ入る。
