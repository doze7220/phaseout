あなたは優秀なフロントエンドエンジニアです。
パズルゲーム『Phase Out: Cluster String』において、スマートフォン画面（縦長）でスコアの表示桁数が増えた際、数字が改行されてレイアウトが崩壊するバグを修正してください。

【現状の問題】
・最大21桁のスコアと拡大された漢字単位（万・億など）を表示した際、ヘッダー領域やリザルト画面の横幅に収まりきらず、自動改行が発生して表示が崩れている。

【修正要件（style.css / score.js等の改修）】
以下の仕様を満たすようにCSS（および必要ならHTMLのラッパー構造）を改修してください。

1. 自動改行の完全禁止
・スコアを表示するコンテナ要素（ヘッダーのスコア領域、およびリザルト画面のスコア領域）に対して、`white-space: nowrap;` を適用し、絶対にテキストが折り返されないようにしてください。

2. 画面幅（コンテナ幅）に応じた自動縮小（Fit-to-Width）
・スコアのテキストが親要素（画面幅）をはみ出す場合、はみ出さずに1行に収まるようにフォントサイズを動的に縮小する仕組みを導入してください。
・【推奨アプローチ】モダンCSSの Container Queries (`container-type: inline-size;`) と `cqi` / `cqw` 単位を使用し、`clamp()` 関数を用いて「最大フォントサイズ」と「コンテナ幅に応じた縮小サイズ」を定義する。
・または、Flexboxの `min-width: 0` と `transform: scale()` を組み合わせるなど、レイアウト崩れを防ぐ最も堅牢な方法を採用してください。

3. 漢字単位（万、億など）のレイアウト保護
・現在の `score.js` で生成している各ブロック（数字＋単位）の構造において、親要素のフォントサイズが動的に縮小されても、右下に配置された単位の「相対的な位置（下揃え、右寄せ）」が崩れないように `em` 単位やFlexboxの `align-items: baseline` などを駆使してCSSを堅牢にしてください。
・必要であれば、`score.js` の吐き出すHTML構造（`<span>`の入れ子など）を、よりCSSで制御しやすい形にリファクタリングしても構いません。

【確認事項】
・改修後、21桁（例： 123 4567 8901 2345 6789）を表示した際でも、スマホ画面内で1行に美しく収まり、文字がコンテナの横幅に合わせて適切に縮小されることを担保してください。
