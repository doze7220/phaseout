パズルゲーム『Phase Out: Cluster Stirring』において、改修のStep2として「スコア・経験値（EXP）の獲得ロジックの全面刷新」および「旧経験値ロジックの削除」を行います。
`PROJECT_ARCHITECTURE.md`と`PROJECT_FUNCTION_INDEX.md`のルールに従い、以下の仕様を実装してください。表示（UIアニメーション）は既存の仕組みに委ね、本ステップでは純粋な内部計算の更新に集中してください。

【対象ファイル】
・config.js (レベルアップ用定数の追加、状態変数の追加)
・logic.js (スコア・経験値・レベルアップ計算の改修、不要コードの削除)

【実装要件】
1. config.js への定数追加と状態定義
・レベルアップに必要な経験値カーブを制御するため、以下の定数を追加してください。
  ```javascript
  export const LEVEL_CONFIG = {
      BASE_REQUIRE_EXP: 10,        // Lv1→2に必要な初期経験値
      EXP_CURVE_MULTIPLIER: 1.5,   // レベルアップごとの必要経験値の増加倍率
  };
・GameState に、各色の累計破壊数を保持するオブジェクト colorDestroyCounts: {} を追加してください。
不要な旧・経験値獲得ロジックの完全削除 (logic.js) ・過去に実装されていた「獲得スコアに応じた経験値の加算」および「LIFEの超過回復（オーバーフロー）に応じた経験値の加算」に関する計算コードを完全に削除してください。
スコア獲得ロジックの刷新 (logic.js の finalizeDestruction 等) ・獲得スコアの計算式を刷新し、低レベルの大連鎖よりも、高レベルの小連鎖の方が稼げるようにします。 ・計算式: 獲得スコア = ベーススコア(Level) × チェイン補正 ・チェイン補正: BigInt(chainCount - 2) ** 2n （※チェイン数が2以下の場合は 1n としてエラーを防ぐこと） ・ベーススコア: 現在の level に応じて指数関数的に増加するBigInt値（例: 10n ** BigInt(level) や適宜調整された式）として計算してください。
経験値（EXP）獲得ロジックの刷新 (logic.js) ・獲得EXPは「破壊した宝石の種類と数」のみをソースとします。以下の2つの減衰ロジックを適用した値を算出し、内部の exp に加算してください。
　A. 大チェイン減衰: 　　今回のチェイン数に応じたベースEXPを以下の式で算出します。 　　ベースEXP = Math.round( chainCount * ( 100 / (chainCount + 100) ) )
　B. 色別の獲得減衰（均等な撹拌の強制）: 　　現在アンロックされている色の中で、「GameState.colorDestroyCounts において最も累計破壊数が少ない色」の破壊数を『基準値』とします。 　　破壊された色ごとの最終獲得EXPを、(基準値 / その色の累計破壊数) の割合で減衰させて加算してください。（※初期状態や破壊数0の場合は100%として扱うなど、ゼロ除算に注意すること） 　　最後に、破壊された色の colorDestroyCounts に今回の破壊数を加算して更新してください。
新色登場時の下駄補正とレベルアップ計算 (logic.js) ・レベルアップ処理の際、次レベルの必要経験値 (nextLevelExp) を以下の計算式で更新してください。 nextLevelExp = Math.floor( LEVEL_CONFIG.BASE_REQUIRE_EXP * ( LEVEL_CONFIG.EXP_CURVE_MULTIPLIER ** ( level - 1 ) ) ) ・レベルアップによって落下する宝石の種類（色）が新たにアンロックされた際、その新色の colorDestroyCounts を0からスタートさせるのではなく、**「現在解放されている既存色の累計破壊数の平均値」**を初期値として設定（下駄を履かせる）してください。
【遵守事項】 ・スコア計算はインフレ（1e+25等の指数表記）を防ぐため、必ず BigInt を用いて計算してください。 ・UI（画面表示）のアニメーションはStep1で実装済みの displayExp に追従させるため、ここでは純粋に GameState.exp, GameState.level, GameState.actualScore の即時更新のみを行ってください。
