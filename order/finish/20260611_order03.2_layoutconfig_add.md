# マジックナンバーの廃止とレイアウト定数（LAYOUT_CONFIG）の導入

あなたは優秀なゲームエンジニアです。
`PROJECT_ARCHITECTURE.md`と`PROJECT_FUNCTION_INDEX.md`の資料を把握し、取り組んでください。
『PHASE OUT: Cluster Stirring』のコードベースにおいて、各UIやSceneファイルに直書きされているレイアウト関連のマジックナンバー（座標、サイズ、マージン、フォントサイズ等）を廃止し、定数として一元管理するリファクタリングを行います。

## 1. config.js の拡張と【コメントの厳守】
`js/core/config.js` 内に、UI・レイアウト専用の定数オブジェクト `LAYOUT_CONFIG` を新設してください。
このファイルは開発チームの人間が頻繁に直接数値を調整するため、可読性が命です。定数を定義する際は、**必ず以下の情報をJSDocまたはインラインコメントで明記することを厳守**してください。
- **なんのUIの数値なのか？**（例: 「タイトル画面のSTARTボタン」「コンフィグ画面の更新履歴エリアの幅」等）
- **どこを基準座標（アンカー）にしているか？**（例: 「画面中央 (WIDTH / 2) を基準」「画面右下端から -20px を基準」等）

以下の要素をカバーする階層的なオブジェクトとして定義すること。
- **BASE**: 基準となる画面の論理解像度（例: `WIDTH: 720, HEIGHT: 1280`）や基準マージン。
- **BUTTON**: ボタンの標準サイズ（幅、高さ）、角丸の半径、標準フォントサイズなど。
- **MODAL**: `UI.Window` 等で用いるモーダル画面のサイズ（幅、高さ）、配置基準座標、タイトルバーの高さなど。
- **TEXT**: 標準テキスト、ヘッダーテキストなどのフォントサイズや行間設定。

## 2. マジックナンバーの置き換え（リファクタリング）
プロジェクト内の以下の主要ファイルを走査し、レイアウトに関する直書きの数値（マジックナンバー）を `LAYOUT_CONFIG` を参照するように書き換えてください。
- `js/scene/TitleScene.js` (テキストやボタンの配置座標、サイズ)
- `js/scene/ConfigScene.js` (ウィンドウサイズ、トグルスイッチの配置座標、スクロールエリアのサイズ等)
- `js/render/UIComponents.js` (デフォルトのマージンやパディング、角丸の `radius` 指定など)
- 必要に応じて `ScoreRenderer.js` や `GaugeManager.js` のUI部分。

## 3. 要件と注意事項
- 既存のレイアウト（見た目や配置）自体は一切変更しないでください。現在表示されている状態の数値をそのまま抽出して定数化することが目的です。
- 定数化の際、画面中央揃えなどは `(LAYOUT_CONFIG.BASE.WIDTH - LAYOUT_CONFIG.BUTTON.WIDTH) / 2` のように、基準値からの相対計算で記述するようにリファクタリングしてください。
