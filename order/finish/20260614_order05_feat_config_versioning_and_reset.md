# ローカル設定のバージョニングとアップデート時の自動リセット機能の実装

現在、`localStorage` に保存された設定データを読み込む際、バージョン間の互換性チェックが行われていません。今後のアップデートで設定項目が変更された際に古いキャッシュでゲームがクラッシュするのを防ぐため、設定データにバージョン情報を記録し、本体バージョンと異なる場合は初期設定へリセットする機構を実装してください。

## 1. config.js の改修
`config.js` 内（`AppConfig` の上部など）に、現在の設定データのバージョンを定義する定数を追加してください。
（例：`AppConfig.CONFIG_VERSION = 'v0.18.2';`）

## 2. 設定の読み込み・初期化ロジックの改修
`localStorage` から設定を読み込む処理を以下のように改修してください。

1. 読み込んだデータに `version` プロパティが存在するか確認する。
2. 存在しない、または `version` が現在の `AppConfig.CONFIG_VERSION` と一致しない場合は、「古いデータ（または初回起動）」とみなし、読み込んだデータを破棄する。
3. その場合、前回のタスクで実装した `AppConfig.DEFAULT_SETTINGS.PC` または `AppConfig.DEFAULT_SETTINGS.MOBILE` を用いて、端末に合わせた初期設定を適用する。

## 3. 設定の保存ロジックの改修
設定を `localStorage` に保存（JSON化）する際、必ず現在の `AppConfig.CONFIG_VERSION` を `version` プロパティとして一緒に記録するように改修してください。

## 4. ドキュメントの更新
実装完了後、グローバルルールに従い `changelog.js` へ今回の改修（v0.18.2）を追記し、関連ドキュメントのバージョン表記を同期してください。
