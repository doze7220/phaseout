# プリズムリンク専用カタルシス演出（UI・エフェクト・サウンド）の実装

本計画書は、プリズムリンク発生時にプレイヤーのカタルシスを最大化する「パチスロ風カウントダウンUI」「レーザー膨張」「重低音SE（ピッチ上昇）」の実装方針を定義するものです。また、マジックナンバーを排除し、各種設定値を分離・一元化します。

## 1. User Review Required

ご提示いただいた設計上の懸念点および追加指示について、以下の通り設計アプローチを提案いたします。ご確認をお願いいたします。

> [!IMPORTANT]  
> **1. `LaserEffect.js` が責務を汚染しないための設計アプローチ**
> 現在 `LaserEffect.js` の `animateLaserLevels` メソッドには、すでにFacadeから `screenEffects` と `playSE` が引数としてDI（依存性の注入）されています。
> したがって、`LaserEffect.js` 内で新たなインポート（結合）を増やすことなく、`nextLevel()` のループ内で Pリンク（`from.colorId !== to.colorId`）を検知した瞬間に、既存の引数を利用して以下のように呼び出します。
> ```js
> // Pリンク検知時
> if (screenEffects) screenEffects.triggerPrismLinkStep(pLinkCount);
> if (playSE) {
>     const pitchRate = Math.min(2.0, 1.0 + (pLinkCount * 0.05));
>     playSE('PRISM_LINK_BURST', { playbackRate: pitchRate });
> }
> ```
> これにより、純粋なレーザー進行管理モジュールとしての責務を保ちながら、上位層である `ScreenEffects` や `SoundManager` のロジックへ安全に委譲することができます。

> [!IMPORTANT]  
> **2. パチスロ風の点灯演出 (Canvas アニメーション) の骨子とコンフィグ分離**
> 追加指示の通り、マジックナンバーを排除し定数へ分離します。
> 
> **状態管理:**
> - 連鎖中に初めてPリンクが発生した瞬間に `active = true` となり、画面上部に描画枠（グリッチ付き）が出現します。
> - `triggerPrismLinkStep(step)` が呼ばれるたびに、対象ステップ（1〜7）の `startTime` を記録します。
> 
> **描画ループ (`drawPopups`等での処理):**
> 1. `LAYOUT_CONFIG.PRISM_LINK_UI` の値を用いて、画面中央に7つのアイコンが均等に並ぶよう動的にX座標を計算（`(画面幅 - 全体の幅) / 2` 等）します。
> 2. 対象ステップの経過時間（`elapsed`）を算出し、`EFFECT_MATH_CONFIG.PRISM_LINK` の定数を参照してアニメーションを制御します。
> 3. `0 〜 DROP_DURATION_MS`: トライバルアイコンを `MAX_SCALE` から等倍（1.0）へ加速度的に縮小させながら描画します。
> 4. `DROP_DURATION_MS 時点`: インパクトの瞬間として `effects.spawnBurstSparks()` で火花を散らし、`FLASH_DURATION_MS` の間白くフラッシュさせます。
> 5. それ以降: 正規の陣営色で点灯状態として定着させます。
> 6. 連鎖終了の `hideChainPopup` と同時に全体をフェードアウトさせます。

  - `DROP_DURATION_MS`: スタンプ落下の所要時間（例: 400）。
  - `FLASH_DURATION_MS`: 落下後の白フラッシュの持続時間（例: 100）。
  - `MAX_SCALE`: 落下開始時の最大スケール（例: 3.0）。
  - `LASER_WIDTH_MULT`: レーザー膨張時の太さの加算倍率（例: 0.5）。

### [Goal Description]

プリズムリンク発生時の専用カタルシス演出（完全版）を実装します。既存の黒背景を廃止したソリッドなパチスロ風スタンプUI、個別のレーザーが保持する「プリズム深度」による膨張と重複排除、ピッチ上昇する専用SE、およびスコア確定時のグリッチ消去アニメーションを追加します。

## User Review Required

> [!IMPORTANT]
> **オープンクエスチョンに対する設計骨子の提示**
> 実装に着手する前に、以下の設計案について承認をお願いいたします。
> 
> **1. 「プリズム深度」を用いた `maxPrismDepth` の状態保持の場所**
> - **提案**: `GameState` ではなく、**`LaserEffect.js` 内のローカル変数（クロージャ）** および **`ScreenEffects.js` の `prismLinkState` 内** に留める設計とします。
> - **理由**: プリズム深度はスコア計算等のコアロジックには影響を与えない「演出専用のステート」。`LaserEffect` の `animateLaserLevels` 内で各宝石の深度をMapで追跡し、新深度に到達した瞬間（`maxPrismDepth` 更新時）に `ScreenEffects` へ通知することで、グローバルな `GameState` を汚染せず、かつ自己完結したクリーンな設計となります。
> 
> **2. スコア確定時の「グリッチ消去アニメーション」のCanvas描画アプローチ**
> - **提案**: DOMは一切使わず、**Canvasの再描画によるピクセルベースのグリッチ** を実装します。
> - **詳細**: `ScreenEffects.js` の `hideChainPopup` 呼び出し時に、ゲージのステートを「フェードアウト」から「グリッチ消滅（`isGlitching = true`）」へ変更します。描画ループ内でこのステートを検知した場合、数フレーム（約150〜200ms）の間のみ、以下の描画処理を適用して瞬時に消滅させます。
>   - ランダムなX/Y座標のシフト（ジッター）
>   - RGBずらし（加算合成による赤色と水色の描画ズレ）
>   - 横方向のスリット（`globalCompositeOperation = 'destination-out'` による部分的な欠け、あるいは矩形によるノイズ）
