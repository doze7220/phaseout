あなたは優秀なゲームエンジニアです。
パズルゲーム『Phase Out: Cluster Stirring』の撹拌ビジュアライザ（Visualizer.js）について、ディレクターからの指示と共有されたデザイン案に基づき、「BLOCKモード（LITE版）」の描画ロジックを「横向きの効率メーター」として全面改修してください。

【対象ファイル】
・js/render/Visualizer.js

【実装要件】
1. レイアウトと動的分割（色数への適応）
・BLOCKモードの描画において、Canvasの高さ（または指定の描画領域）を「現在アンロックされている色の数（`activeColors.length`）」で均等に分割してください。
・上から順に、`activeColors` の配列順（赤、青、緑、黄…）で各色のメーターを横向きに描画します。
・レベルアップにより新しい色（陣営）がアンロックされた際は、自動的に各メーターの縦幅（太さ）が細くなり、一番下に新色のメーターが追加される計算ロジックにしてください。

2. X軸（長さ）への「経験値獲得効率」マッピング
・各色のメーターの基本の長さ（横幅）を、左端（0%）から右端（100% = CanvasWidth または任意のマックス幅）への「経験値獲得効率」として描画してください。
・計算式: `効率(0.0〜1.0) = (全色の中で最も少ない累計破壊数) / (その色の累計破壊数)` （※ゼロ除算に注意し、破壊数0の場合は1.0とする）

3. 脈動とスパイク（跳ね上がり）の表現
・**常時の脈動:** メーターの長さに対して、時間（`time` や `Date.now()`）をソースとした複数のサイン波を合成し、「常に±5%程度のランダムな揺らぎ（呼吸のような脈動）」を加えてください。
・**スパイク:** その色が破壊された瞬間の `visualizerAmplitudes[color]`（スパイク倍率）の値をメーターの長さに瞬時に加算し、「一瞬だけ右に鋭く跳ね上がり（ビュッと伸びて）、すぐに元の長さに戻る」というリアクティブな動きを実装してください。

4. 描画スタイル（スリット入りバー表現）
・デザイン案に寄せるため、単なる1つの大きなベタ塗り四角形（`fillRect`）ではなく、「細い横線（スリット）が複数重なったバー」または「細かいブロックが連続したバー」のように描画してください。
・LITE版の軽量さを維持するため、高負荷な加算合成やパス描画は極力避け、`fillRect` のループ処理等でサイバーな計器風デザインを構築してください。
・背面のUIとしての役割を果たすため、前面のスコアやタイマーのテキストの視認性を損なわないよう、透明度（`globalAlpha`）や色味を適切に調整してください。

【厳守事項】
・WAVEモードのロジックには影響を与えず、コンフィグ（`AppConfig.VISUALIZER_MODE`）で 'BLOCK' が選択された時のみこのメーターが描画されるようにしてください。
