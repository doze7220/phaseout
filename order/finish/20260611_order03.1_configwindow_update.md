# ConfigSceneの完全実装とモーダルUXのモダナイズ

あなたは優秀なゲームエンジニアです。
`PROJECT_ARCHITECTURE.md`と`PROJECT_FUNCTION_INDEX.md`の資料を把握し、取り組んでください。
現在スタブとなっている `js/scene/ConfigScene.js` を本格実装し、古いDOMベースのコンフィグモーダルを完全に駆逐します。同時に、モーダルのUXを最新のトレンドに合わせて改修します。

## 1. モーダルUXのモダナイズ (UI.Window / ConfigSceneの連携)
最新のトレンドUXを採用するため、以下の動作を実装してください。
- **枠外タップで閉じる:** コンフィグモーダル展開時、ウィンドウの矩形外（半透明の暗幕部分）がタップされた場合は、即座に `SceneManager.popScene()` を呼び出してモーダルを閉じる仕組みを実装してください。
- **右上のXボタンのみ:** 従来のウィンドウ下部にあった「閉じる」テキストボタンは廃止します。ウィンドウの右上にのみ、既存のアセット（`btn_close.png` 等）を利用した `UI.ImageButton` の「Xボタン」を配置し、タップで `popScene()` を実行させてください。

## 2. ConfigScene.js の本格実装
- `UI.Window` を用いて、美しいコンフィグモーダルを構築してください。
- 現在のDOMコンフィグが持っている以下の要素を、Canvas UIコンポーネントを用いて再構築してウィンドウ内に配置してください。
  1. スコア表示形式トグル (`UI.ToggleSwitch` 等)
  2. エフェクト設定切替 (FULL/LITE/NONE の切り替えUI)
  3. 更新履歴 (ChangeLog) の表示 (`UI.ScrollArea` を使用)
- 描画は `PROJECT_ARCHITECTURE.md` の定めに従い、第9層（MODAL_UI）のタイミングで行うこと。

## 3. main.js とパズル画面の統合＆クリーンアップ
- `main.js` 等に残存している古いコンフィグ制御ロジック（`closeConfigModal` 関数や関連するDOMイベントリスナー）を完全に削除してください。
- パズル画面での「コンフィグボタン（歯車）」タップ時の処理を、古いDOM呼び出しから `SceneManager.pushScene(new ConfigScene())` へと置き換えてください（すでにPlayScene等で実装されている場合は連携を確認してください）。
- `index.html` と `style.css` から、古いコンフィグ用のDOM要素（`#config-modal` 等）を完全に削除し、不要なCSSをクリーンアップしてください。
