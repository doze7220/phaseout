現在開発中のパズルゲーム『Phase Out: Cluster String』のアーキテクチャ資料（`architecture.md`）を、超詳細かつ厳密な設計図へと全面的にアップデートしてください。

【目的】
今後の機能拡張（エージェントのスキル実装、プリズムリンクなど）や、AIエディタとのコード共有において、バグや先祖返りを防ぐための絶対的なルールブック（Single Source of Truth）を作成すること。

【構成と必須項目】
以下の構成に従ってMarkdown形式で見出しや表（テーブル）を用いて視認性高く出力してください。

#### 1. プロジェクト概要と駆動方式
・ランタイム: Vanilla HTML + JavaScript + Canvas 2D API + DOM(CSS)
・物理エンジン: Matter.js
・ゲームループが `requestAnimationFrame` と Matter.js の `Engine.update` によってどのように協調して駆動しているか（ハイブリッド駆動）を定義してください。

#### 2. ファイル構成と各ファイルの厳密な責務
・現在の `architecture.md` にあるファイル群（config.js, physics.js, logic.js, effects.js, renderer.js, score.js, main.jsなど）を、「データ定義」「システム・ロジック」「レンダリング・UI演出」などの論理的なカテゴリに分類し、それぞれの「責務（何をするか）」と「やらないこと」を一覧化してください。

#### 3. Game State（状態管理）一覧
・ゲームの状態機械として以下を定義してください。
  - TITLE（タイトル画面）
  - PLAYING（通常プレイ中）
  - STASIS（一時停止中：コンフィグメニュー展開時、またはライフ0の演出時など。Matter.jsのtimeScale制御と連動）
  - RESULT（リザルト画面）
  - PHASE_BREAK（※将来実装予定：7色連鎖時のフィーバー状態）

#### 4. ★絶対変更禁止：Update・イベントフック順序
・Matter.jsのイベントフック（`Events.on`）とゲームロジックの実行順序を厳密に定義してください。
  1. `beforeUpdate` (タップ入力の処理、引力の適用など)
  2. 物理エンジンのステップ実行
  3. `collisionStart` (宝石の衝突判定、連鎖起点の発火など)
  4. `afterUpdate` (連鎖BFS探索、パーティクル座標の更新、スコア計算、LIFE増減など)

#### 5. ★絶対変更禁止：Draw順序とUIバインディング規則
・描画とUI更新の順序を定義してください。
  - **Canvas描画順**: 1. 背景(星空等) → 2. 宝石オブジェクト（カスタムレンダラー） → 3. レーザー・伝播線（加算合成） → 4. 破片・火花パーティクル → 5. 波紋・フラッシュ演出
  - **DOM/CSSバインディング規則**: JavaScriptのデータをDOMに反映させるルール（例：ドラムロール中の `displayScore` を毎フレームDOMにテキスト出力する、トップ破壊色のカラーコードをCSS変数 `--aura-color` に流し込む、LIFEゲージをSVGの属性変更で追従させる等）。

#### 6. リファクタリング共通ルール（注意事項）
・本プロジェクト特有の開発ルールを列挙してください。
  - 「Matter.jsの物理座標（Body.position）を描画ループ内で直接書き換えない（副作用禁止）」
  - 「DOMへのアクセスは必要最小限に抑え、描画負荷を下げる」
  - 「Canvasの加算合成（`lighter`）を使用する際は、必ず `save()` と `restore()` でコンテキストを保護し、他の描画に漏らさない」
  - 「状態管理（スコアやレベル）は必ず一元化された変数から参照する」
