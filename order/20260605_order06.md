あなたは優秀なゲームエンジニアです。
パズルゲーム『Phase Out: Cluster String』において、リザルト画面（ゲームオーバー時の結果表示）の表示項目を大幅に拡充し、アニメーションのテンポを最適化してください。
アーキテクチャ資料（`architecture.md`）の規定に従い、状態管理、集計ロジック、UI演出の責務を正しく分割して実装してください。

### 【1. GameStateと集計ロジックの拡張（config.js / logic.js）】
・`config.js` の `GameState` に以下のプロパティを追加し、初期化処理に含めてください。
  - `playStartTime`: プレイ開始時のタイムスタンプ
  - `maxChain`: 1プレイ中の最大連鎖数
  - `maxScorePerTap`: 1タップから派生した連鎖で獲得した最大のスコア（BigInt）
  - `maxChainPerColor`: 各色ごとの最大連鎖数を保持する配列（`disrupts` と同じ構造）
・`logic.js` 内で、連鎖終了後（スコア加算時）に上記の `maxChain`、`maxScorePerTap`、および各色の最大連鎖数を更新するロジックを追加してください。
・プレイ開始時（PLAYING移行時）に `playStartTime` を記録してください。

### 【2. リザルト画面のレイアウト改修（index.html / style.css）】
・リザルト画面（`#result-screen`）のレイアウトを拡張し、「メインリザルト領域」と「詳細ログ領域」に分割してください。
・**メインリザルト（大きく表示）:**
  1. 最終スコア（従来のフォーマット）
  2. 最終到達レベル
  3. プレイ時間（MM:SS 形式）
  4. 最大チェイン数
  5. 1タップの最大獲得スコア
・**詳細ログ（下部にリスト表示）:**
  - 総破壊数（全色の合計）
  - 各色ごとの「破壊数」および「最大チェイン数」を並べて表示するリスト。

### 【3. アニメーションとタップ制御（ScreenEffects.js）】
・`ScreenEffects.js` の `showResult` メソッドを改修し、以下のタイムラインで要素を順番に表示（フェードイン）させてください。
  - **0.0秒**: 背景・枠組み表示（この瞬間から画面タップによるタイトル遷移を無効化）
  - **0.5秒**: 最終スコア を表示
  - **1.0秒**: 最終到達レベル を表示
  - **1.5秒**: プレイ時間 を表示
  - **2.0秒**: 最大チェイン数 を表示
  - **2.5秒**: 1タップの最大獲得スコア を表示
  - **3.0秒**: 詳細ログ領域（総破壊数、各色のデータ）を**一括で表示**
  - **3.0秒以降**: 画面タップによる遷移の**制限を解除**（タップでタイトルへ戻れるようにする）

### 【実装時の注意事項】
・表示項目の増加に伴い、スマートフォンの画面（論理サイズ 720x1280）に収まるよう、CSSでフォントサイズやマージン、Flexbox/Gridを適切に調整してください（必要であれば詳細ログ領域はスクロール可能にしても構いません）。
・スコアのテキスト出力には、既存の `score.js` のフォーマット関数を活用してください。
・物理演算（Matter.js）や既存の描画ループに影響を与えないよう注意してください。
