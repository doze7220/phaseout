あなたは優秀なゲームエンジニアです。
以下の仕様書に基づき、ブラウザ上で動作する「物理演算を用いた連鎖パズルゲーム」のプロトタイプを作成してください。
実行確認が容易にできるよう、HTML、CSS、JavaScript（Matter.jsを使用）をすべて1つにまとめた `index.html` ファイルとして出力してください。物理エンジンはCDNから `matter.js` を読み込んでください。

### 【ゲームの基本仕様】
・Matter.js を使用し、物理演算ベースの非グリッド型パズルを作成する。
・画面中央に幅400px、高さ600pxの枠（容器となる壁と床）を作成し、標準的な下向きの重力を設定する。

### 【オブジェクト（宝石）の仕様】
・形状: 円形（半径 15px 〜 20px のランダム）
・色: 7種類（0〜6の colorId を割り当て、それぞれ視認しやすい異なる色で描画する）
・物理プロパティ: 摩擦(friction): 0.05、反発係数(restitution): 0.3、密度(density): 0.001
・ゲーム開始時、枠内に十分な数の宝石を上部から降らせ、物理演算で敷き詰める。

### 【コアロジック：BFS連鎖探索と破壊】
・プレイヤーが宝石をクリックした際、以下の処理を行う。
1. クリックされた宝石の `colorId` を取得する。
2. クリックした宝石と物理的に「接触（Collision）」している同色の宝石を、幅優先探索（BFS）を用いて再帰的に検索し、連鎖リストを作成する。
3. 連鎖リストの個数 `n` に応じて以下の処理を行う：
   - n >= 3 の場合：対象の宝石をすべてワールドから削除する。
   - n < 3 の場合：対象の宝石を削除するが、スコアは加算しない。

### 【スコア計算とUI】
・画面上部にスコアを表示する。
・n >= 3 で削除した場合、スコアを加算する。
  加算式の計算ロジック: `score += Math.floor(Math.pow(10, (n - 3) * 0.5) * 100)`
・スコア加算時は、表示テキストを即座に更新する。

### 【補充システム】
・宝石が削除された場合、削除された個数と同数の新しい宝石を、枠の上部（y座標が0以下の画面外）のランダムなx座標から生成し、落下させる。生成される色は7色からランダムとする。

### 【演出（可能であれば）】
・消滅する条件を満たした場合、対象となる宝石をすぐに消すのではなく、対象の宝石同士を中心点で結ぶ「光の線」を0.2秒ほど描画してから削除する演出を入れる。
