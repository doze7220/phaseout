# 階層（Depth）ボーナスの導入と数式リアルタイム可視化UIの実装

## Goal
瞬時のプレイヤーの判断がスコアに直結する要素を入れ、このパズルへの習熟度・プレイスキルとしてプレイヤー自身の成長要素を提供するため、パズルにおける「消し方の形（長く繋ぐか、丸く固めるか）」を評価する「階層数（Depth）」の概念を基本ルールの計算に組み込みます。
同時に、その計算式がリアルタイムに跳ね上がっていくカタルシスを演出する「数式ポップアップUI」を実装し、コンフィグからON/OFFを設定できるようにします。

## 1. 仕様詳細

### 1-1. 階層数（Depth）の取得と基本スコア計算への組み込み
- `logic.js` の `startChain` 内で行われているBFS探索に手を入れて、連鎖の「最大階層数（maxDepth）」を算出・保持してください。
- 獲得スコアの基本計算式を、全プレイヤー共通の仕様として以下のようにアップデートします（BigIntで計算すること）。
  `RATE * (連鎖数 - 2) ^ 2 * (1 + (階層数 / 10))`
- 上記の「分母の10」はマジックナンバーにせず、`config.js` の `CORE_MATH_CONFIG` に `DEPTH_BONUS_DIVISOR: 10n` として定義してください。

### 1-2. UI演出（コンフィグによるON/OFF切り替え）
- `config.js` に新たに `SHOW_MATH_POPUP: true` （デフォルトON）を追加してください。
- 連鎖演出時のポップアップ表示を以下の2段構えに改修します。
  - **ベースUI（常時表示）:**
    従来の「XX Chain」という表示を、「Chain XX / Depth XX」という並記スタイルに変更します。
  - **詳細数式UI（`SHOW_MATH_POPUP` が `true` の場合のみ追加表示）:**
    ベースUIの上に被せる、または上部にスライドインさせる形で、「Score」と「RATE * XX² * 1.XX * [ボーナス倍率]」がリアルタイムにカウントアップする専用の数式UI（キャンバスまたはDOM）を描画します。
    ※ 【重要】連鎖ボーナスの部分は「^ 2」ではなく、実際の数式のように上付き文字（²）として美しくレンダリングしてください。
    ※ 【重要】`1.XX` の部分は、基本ルールとして算出した「Depthボーナス（例：1.25など）」を常に表示してください。
    ※ 【重要】キャラクターによる編成ボーナス（Densityなど）が適用される場合は、その後ろに `* [× 3.5]` のような形で追加描画してください。未編成時（ボーナスなし）の場合は、この部分は完全に非表示とし、基本式（`RATE * XX² * 1.XX`）のみを描画してください。

### 1-3. カウントアップ時の「Pop＆発光」演出
- 数式UIのカウントアップ中は、陣営色を使わずに数字が切り替わるごとにスケールを1.2倍に弾ませる（Popさせる）アニメーションを入れ、インフレの勢いを表現してください。
- 連鎖が終了し最終スコアが確定した瞬間に、白フラッシュを伴ってガシャン！と陣営色（青や黄など）へ強く発光（グロウ）させる演出を入れてください。

### 1-4. コンフィグ画面への組み込み
- `main.js` のコンフィグモーダル内に、「詳細スコア表示（数式ポップアップ）」をON/OFFできるトグルスイッチを追加し、設定を反映できるようにしてください。

## 2. Proposed Changes

### [MODIFY] config.js
- `CORE_MATH_CONFIG` に `DEPTH_BONUS_DIVISOR: 10n` を追加します。
- AppConfig等のコンフィグ設定群に `SHOW_MATH_POPUP: true` を追加します。

### [MODIFY] logic.js
- `startChain` 関数内のBFS探索ロジックを改修し、起点を `depth = 1`、隣接キュー追加時に `depth + 1` とし、`maxDepth` を取得します。
- 取得した `maxDepth` を用いて、新しい基本数式（BigIntでのパーセンテージ計算）でスコアを算出させます。

### [MODIFY] ScreenEffects.js & ScoreRenderer.js
- `showChainPopup` 関数の表示文字列を「Chain XX / Depth XX」にアップデートします。
- `config.SHOW_MATH_POPUP` が `true` の場合、スコアと数式（`RATE * XX² * 1.XX * [倍率]`）を表示・カウントアップさせるロジックを追加します。
- `ScoreRenderer.js` の文字列描画ロジックに、上付き文字（²）のスケール/Y座標オフセット対応、およびカウントアップ時のPopアニメーション・確定時の陣営色発光対応を組み込みます。
- キャラクター編成ボーナスが無い場合は、倍率部分の描画をスキップする判定を追加します。

### [MODIFY] main.js
- コンフィグメニューのDOM生成・イベントリスナーに「詳細数式ポップアップ（SHOW_MATH_POPUP）」のトグルを追加します。

### [MODIFY] PROJECT_MATH_AND_BALANCE.md
- 「獲得スコア」の計算式を最新の Depth ボーナス込みの数式に更新します。
- UI演出の項目に「数式ポップアップ表示（Pop・確定発光アニメーション含む）」の条件を追記します。
