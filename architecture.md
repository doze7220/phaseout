# ファイル構成と各モジュールの役割（目安：各ファイル最大300〜500行程度に収める）

- `index.html` (約120行): エントリーポイント、各画面（タイトル、ホーム、パズル、ガチャ、リザルト等）のシーンコンテナ・UI定義
- `main.js` (約120行): アプリ全体の初期化、シーン遷移の制御ハブ
- `config.js` (約130行): システム全体で共有する状態管理（`GameState`）とユーザー設定（`AppConfig`）の定義。各種パラメータ、基本色、物理設定などの定数化
- `physics.js` (約150行): Matter.jsのエンジン初期化、壁（境界）の生成、宝石オブジェクトの物理パラメータ設定とスポーン処理
- `logic.js` (約220行): タップ入力の検知、BFS（幅優先探索）による連鎖判定、ライフ増減のロジック、ゲームオーバーおよび盤面レベルの管理
- `effects.js` (約420行): レーザーや火花などのパーティクル演出、ピンチ・ステイシス時の画面効果（ヴィネット等）、UIアニメーション制御
- `renderer.js` (約240行): Matter.jsの描画ループに割り込むカスタムレンダラー。キャンバスキャッシュの生成管理と、宝石の拡縮・フラッシュ・沈み込み・脱色等の視覚効果を担当
- `scene.js` (約150行): シーン遷移ロジックの実装。リザルト画面のUI制御とスコア表示、再挑戦（リトライ）の処理
- `score.js` (約40行): スコアの桁区切りやフォーマットなどのユーティリティ
- `title-animation.js` (約190行): タイトル画面における専用の背景アニメーション制御
- `style.css` (約420行): UIレイアウト、オーバーレイエフェクト（ピンチ/ステイシス）、CSSアニメーションのスタイル定義
- `changelog.js` (約360行): 各バージョンの更新履歴の記録・管理
